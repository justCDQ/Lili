# 每日产品或功能拆解

每天选择一个产品中的一个具体功能或任务，完成一次小型产品分析。不要泛泛分析整个产品，也不要把界面截图当成拆解结论。

## 观察路径

```text
目标用户与场景 → 当前问题与替代方案 → 功能方案与流程
→ 用户价值 → 产品目标与指标 → 商业/技术成本 → 风险与改进
```

证据可以来自真实使用、官方文档、定价页、更新日志、公开 Issue、产品评论、状态页、公开数据和竞品对比。始终标记事实、推断与个人观点。

## 记录方式

1. 复制 [_template.md](_template.md)。
2. 按 `YYYY/MM/YYYY-MM-DD-product-feature.md` 保存。
3. 同一产品可以多次出现，但每次只分析一个任务、功能或决策。
4. 每七次拆解做一次横向总结，提炼可迁移的产品模式。

## 完成标准

- [ ] 说明用户、场景、任务和现有替代方案。
- [ ] 至少使用一个带日期和链接的外部证据或真实使用记录。
- [ ] 分析价值、指标、成本和取舍，不只描述功能。
- [ ] 给出一个保留、一个删除或一个改进建议及其风险。
