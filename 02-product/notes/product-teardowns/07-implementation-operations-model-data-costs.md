# 实现成本、运营成本、模型成本与数据依赖

## 是什么

实现成本包括需求、设计、开发、测试、迁移和发布；运营成本包括基础设施、监控、支持、内容审核、合规与维护；模型成本包括输入输出 Token、推理资源、评测、重试和人工复核；数据依赖包括采集权、质量、时效、覆盖、许可、隐私、存储与删除。

## 为什么需要

只估开发工时会使上线后持续成本不可见。模型调用、人工审核和数据更新通常随使用量增长，可能让一个有用户价值的方案在规模扩大后不可持续。

## 关键特性或判断规则

- 分开一次性、固定持续和随用量增长的变量成本。
- 估算必须有单位，例如每请求、每任务或每活跃用户。
- 数据依赖检查权利、质量、时效、覆盖、删除和纠错。
- 对不可观察的内部实现只给区间和假设，不写成事实。

## 实际怎么使用

不猜具体架构，按可观察能力建立成本驱动表：`能力｜一次性工作｜随用户/请求增长的变量成本｜固定成本｜人工环节｜外部依赖｜失败影响｜证据/推断`。估算低中高区间，并写单位经济：每任务、每活跃用户或每交易成本。

对 AI 功能额外记录上下文长度、输出长度、缓存、并发、延迟目标、评测集和人工接管。对数据功能记录来源、更新频率、删除和纠错机制。

## 常见错误与边界

- 将第三方 API 价格当全部成本。
- 忽略支持、审核、迁移和停用成本。
- 用前端表现断言后端实现。
- 公开定价会变化；分析具体价格必须注明观察日期和区域。

## 补充知识

边际成本低不表示总拥有成本低。上线前还需估算迁移、监控、供应商锁定、停用和故障恢复；AI 方案应将评测和人工复核视为持续运行成本。
