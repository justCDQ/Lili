# 默认值、限制、渐进披露与范围取舍

默认值决定用户不主动选择时产品怎样行为；限制定义容量、权限、速率、格式和商业边界；渐进披露决定什么时候展示哪些信息；范围取舍决定当前版本完成哪些场景、明确不处理哪些场景。四者共同影响任务是否可发现、可完成、可控制和可恢复。

## 默认值

默认值不是中性起点。多数用户不会修改所有设置，因此默认会实际分配隐私、费用、通知、权限和风险。

每个默认值记录：

| 字段 | 问题 |
| --- | --- |
| 适用对象 | 新账号、旧账号、某套餐或某角色？ |
| 初始值 | 未操作时实际生效什么？ |
| 告知 | 用户在何时看到影响？ |
| 修改 | 谁可以改，入口在哪里？ |
| 生效范围 | 当前对象、未来对象或全部历史对象？ |
| 迁移 | 默认改变后旧对象怎样处理？ |
| 恢复 | 能否撤销，已有数据怎样处理？ |

### 安全和隐私默认

涉及公开分享、数据处理、付费、通知和高权限时，默认需要保守、透明且可改变。是否符合法律要求取决于具体地区和处理情境；产品拆解至少要检查数据最小化、访问、说明、关闭和删除。

### 默认改变

新版本改默认时，必须区分新建对象与已有对象。静默追溯改变旧权限或旧数据处理会产生高风险。测试要使用新账号、升级账号和历史对象。

## 限制

限制可以来自技术可靠性、成本控制、安全、业务规则、套餐包装或法律政策。没有内部证据时，只能记录产品公开或可观察限制，不能断言设计原因。

### 限制契约

每项限制说明：

- 数值和单位，例如每文件 100MB；
- 计量对象，例如文件、组织、用户或请求；
- 时间窗口和重置时间；
- 触发前能否发现；
- 达到限制时是拒绝、排队、降级还是收费；
- 已完成工作是否保留；
- 用户可以等待、缩小、升级、导出或转人工；
- 临界值和并发怎样处理。

“上传有限制”不可测试。“单文件最大 100MB，选择前显示，101MB 被拒且本地文件保留”可测试。

### 技术限制与商业限制

相同界面可能混合两种边界。免费套餐 100MB、付费套餐 1GB 是包装；所有套餐都拒绝损坏文件是质量约束。拆解时分开，避免把升级提示当唯一恢复。

## 渐进披露

渐进披露先显示完成当前任务所需的信息，再在用户选择、状态或复杂度增加时展示高级内容。它用于降低初始认知负担，不用于隐藏费用、风险、错误后果和任务状态。

### 适合后置的信息

- 低频高级格式；
- 专家参数和细粒度配置；
- 不影响当前决定的解释；
- 仅在特定状态出现的恢复选项。

### 必须及时显示的信息

- 价格、自动续费和会产生费用的动作；
- 数据公开、删除、覆盖和不可逆后果；
- 当前用量、剩余额度和限制；
- 错误、等待、异步状态和恢复；
- 权限范围和受影响对象；
- 任务完成所必需的输入。

隐藏并不等于移除。二级内容仍需可发现、键盘可达、屏幕阅读器可理解，并在用户需要前出现。

## 范围取舍

范围以完整任务闭环为单位，而不是以页面或组件数量衡量。最小范围仍需包含入口、前置、核心行为、权限、结果、失败和恢复。

### 三层范围

| 层级 | 含义 | 处理 |
| --- | --- | --- |
| 核心场景 | 首版必须完整完成 | 可靠实现并设结果指标 |
| 边界场景 | 价值成立但暂时覆盖有限 | 明确限制、降级和证据 |
| 非目标 | 当前不承担 | 在入口前说明并提供替代 |

只做“上传按钮”但没有进度、失败和恢复，不是小范围闭环。删除高级压缩可以缩小范围；删除权限校验则破坏任务安全。

## 四者的关系

```mermaid
flowchart LR
    A["任务与目标用户"] --> B["范围：做什么/不做什么"]
    B --> C["默认：未选择时怎样做"]
    B --> D["限制：允许到哪里"]
    C --> E["披露：何时展示选择与后果"]
    D --> E
    E --> F["用户结果与恢复"]
```

范围决定支持的场景，默认决定常见路径，限制保护边界，披露让用户理解这些选择。

## 事实、推断与假设

- 观察事实：新测试账号的云备份默认关闭；帮助写明单文件上限 100MB；
- 推断：限制可能用于成本和可靠性；没有公开证据时不能写成事实；
- 假设：在选文件前展示剩余额度会降低超限失败。

具体产品记录日期、套餐、地区、账号年龄和设备。个人测试证明该环境的行为，不代表全部用户。

## 个人可执行证据

- 新旧测试账号比较默认；
- 帮助、定价、权限和隐私材料确认公开边界；
- 在限制前后值执行任务；
- 使用键盘、窄屏和屏幕阅读顺序检查披露；
- 公开评论、FAQ 和支持材料发现隐性限制；
- 竞品与本地文件、手工导出等非软件替代比较；
- 有授权时分析超限、设置改变、失败与支持事件。

不要为测试使用真实敏感数据，也不要绕过配额或权限。

## 完整案例：云备份设置

### 输入与证据

使用自己控制的新账号、升级后的旧账号和两个设备，准备不含敏感信息的 99MB、100MB、101MB 测试文件。官方帮助说明备份范围、容量和关闭方式；定价页说明套餐容量；隐私材料说明处理类别。非软件替代是本地文件、手工导出和外接存储。

观察事实限定于测试环境。关于默认选择的商业动机保持未知。

### 步骤一：定义核心任务

目标用户需要在设备故障后恢复个人文件。首版核心场景是单账号、支持格式、网络恢复后可在第二设备下载。共享团队备份和长期归档属于非目标。

### 步骤二：审计默认

记录首次启动默认、告知、开启入口和权限。创建备份后关闭，分别检查本地、云端和第二设备状态。再用旧账号检查升级是否追溯改变。

### 步骤三：审计限制

在 99MB、100MB 和 101MB 选择文件；记录单位、何时提示、已完成工作和恢复。测试免费额度耗尽、并发上传、断网、空间不足和不支持格式。

### 步骤四：审计披露

记录首屏显示的备份范围、费用、删除后果、状态和剩余额度。高级压缩可在二级入口；数据公开、付费和失败不能被隐藏。使用键盘与窄屏确认入口可达。

### 步骤五：检查范围闭环

完整路径为开启 → 选择 → 上传 → 可见进度 → 完成 → 第二设备恢复 → 关闭/删除。任何一步没有状态与恢复，都属于范围缺口。

### 输出

输出默认矩阵、限制契约、信息层级和范围卡。事实示例：新账号默认关闭、100MB 成功、101MB 在选择后被拒且本地文件保留。推断与假设单独记录。

### 验证

- 第二设备恢复验证用户结果；
- 刷新后状态仍一致；
- 关闭后确认已有云端副本的处理；
- 99/100/101MB 每项重复三次；
- 超限后已选文件和其他上传不丢失；
- 键盘能到达高级选项；
- 费用和删除后果在决定前可见。

若“提前显示剩余额度”候选在 20 个固定超限任务中把无效提交从 14 次降到 3 次，同时正常上传完成率不下降，可支持该假设；不能用提示曝光量替代任务结果。

### 失败分支

- 新旧账号默认不同：按迁移批次记录；
- 关闭同步但云端保留：必须明确删除与保留；
- 超限后输入丢失：恢复失败；
- 费用提交后才出现：关键信息披露失败；
- 文档按 MB、产品按 MiB：记录单位并复测；
- 弱网显示成功但第二设备无文件：系统或用户完成失败；
- 内部原因不可见：只记录限制行为；
- 核心恢复依赖付费升级：定位和定价边界需明确。

## 非软件替代

本地存储提供最高直接控制但设备损坏时无法恢复；手工导出需要纪律和版本管理；外接存储不依赖网络但可能遗失；人工托管增加信任和费用。用同一恢复任务比较时间、成功、隐私、费用和退出。

## 常见错误

- 把默认当中性；
- 改默认时忽略旧对象；
- 限制到失败后才显示；
- 单位、窗口和重置不清；
- 超限只提供升级；
- 为简洁隐藏费用和风险；
- 高频核心操作被深藏；
- 首版只完成任务前半段；
- 非目标没有失败和替代；
- 根据界面猜测内部原因。

## 结果指标与回归矩阵

设置改动不能只统计修改率。至少观察：

| 指标 | 计算 | 守护问题 |
| --- | --- | --- |
| 默认路径完成率 | 未修改默认且完成任务 / 进入任务 | 默认是否适合目标场景 |
| 主动修改率 | 修改默认的用户 / 看到设置的用户 | 高修改可能表示默认不适合，也可能是探索 |
| 限制前知晓率 | 提交前已看到限制的超限任务 / 超限任务 | 需要可靠曝光事件 |
| 超限恢复率 | 使用合法替代完成的超限任务 / 可恢复超限 | 升级购买不等于原任务完成 |
| 假完成率 | 显示完成但第二设备未恢复 / 显示完成 | 备份任务的关键守护 |
| 关闭后残留率 | 期望删除但仍存在的对象 / 已关闭对象 | 必须按公开保留规则解释 |

默认改变需要回归矩阵：新账号与旧账号、设置未修改与已修改、当前对象与历史对象、免费与付费、Web 与移动。每个格子执行开启、使用、关闭、重新登录和恢复。若某格没有权限或设备，应标为未验证，不用其他格结果代替。

对于渐进披露，测量“能否在需要前找到”比测量“展开次数”更重要。固定任务中记录找到设置的时间、错误选择、是否理解后果和键盘完成。展开少可能表示不需要，也可能表示不可发现，必须结合任务结果解释。

## 拆解检查表

- 默认的对象、告知、修改和追溯是否明确；
- 隐私、安全和付费默认是否可解释；
- 限制是否有数值、单位、窗口和恢复；
- 是否测试临界值；
- 超限是否保留工作；
- 费用、风险、状态是否及时显示；
- 渐进内容是否可发现和可访问；
- 核心范围是否端到端闭环；
- 边界与非目标是否有替代；
- 事实、推断和假设是否分开。

## 练习

选择备份、文件上传或套餐设置，完成默认、限制、披露和范围审计。

验收：使用新旧账号；至少测试一个数值限制的下界、边界和上界；验证一次关闭或删除；使用第二设备确认结果；比较本地或人工替代；输出观察事实、推断与假设；构造一个“操作成功但用户结果失败”的分支。
