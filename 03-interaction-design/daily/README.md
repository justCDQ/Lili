# 每日交互拆解

每天拆解一个具体任务的交互，并尽量画出流程、状态或重新设计。产品拆解关注“为什么做、创造什么价值”，交互拆解关注“用户如何理解、行动、获得反馈和恢复”。两份记录可以互相链接。

## 每次至少观察

- 入口、前置条件、主流程、分支、异常与完成状态。
- 信息层级、视觉焦点、操作层级、反馈和文案。
- 空、加载、成功、失败、无权限、离线、冲突等状态。
- 使用的交互模式、可访问性和工程实现限制。
- 一个重新设计方案及其收益、成本和风险。

## 记录方式

1. 复制 [_template.md](_template.md)，或使用更完整的 [产品拆解模板](../product-teardown-template.md)。
2. 按 `YYYY/MM/YYYY-MM-DD-product-task.md` 保存。
3. 流程图、线框图、截图和原型放在同名 assets 目录，避免散落。
4. 每十次拆解，把重复规律整理进自己的交互模式库笔记。
