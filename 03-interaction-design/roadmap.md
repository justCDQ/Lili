# 交互设计 Roadmap

## 一、基础原则

学习用户目标、心智模型、可见性、反馈、一致性、映射、认知负荷、容错、撤销和渐进披露。

Todo：

- [ ] 分析十个产品的主要任务。
- [ ] 分析十个空状态、错误状态和恢复机制。
- [ ] 分析系统如何告诉用户当前在哪里、可以做什么、正在发生什么。
- [ ] 识别页面要求用户记忆的信息。
- [ ] 区分视觉、交互和产品问题。

验收：给一个页面，能说明用户目标、第一视觉焦点、操作反馈、错误恢复和可以删除的信息。

---

## 二、用户研究与可用性

### 访谈

- [ ] 询问最近一次真实经历。
- [ ] 避免诱导和未来意愿。
- [ ] 记录行为、原话和替代方案。
- [ ] 区分单个意见与重复模式。

### 可用性测试

- [ ] 定义真实任务。
- [ ] 观察用户操作而不是让用户评价稿子。
- [ ] 记录停顿、误操作、求助、完成率和时间。
- [ ] 按阻塞、严重、一般、轻微分类。
- [ ] 根据测试修改设计。

验收：完成三次访谈和一次可用性测试，并能把观察转成设计修改。

---

## 三、信息架构

学习分类、导航、页面层级、搜索、标签、卡片分类、站点地图和渐进披露。

Todo：

- [ ] 绘制一个复杂产品的现有站点地图。
- [ ] 标记重复、混乱和层级过深的入口。
- [ ] 按任务、角色或业务对象重新分类。
- [ ] 设计顶部、侧边、Tab、面包屑和搜索的边界。
- [ ] 找用户测试能否快速找到目标功能。

验收：能对几十个功能建立清晰分类，并解释为什么使用页面、弹窗、抽屉或行内编辑。

---

## 四、用户流程与状态

学习 User Flow、Task Flow、Journey Map、主流程、分支、异常、中断和恢复。

所有功能至少检查：

- [ ] 初始。
- [ ] 空。
- [ ] 加载。
- [ ] 成功。
- [ ] 失败。
- [ ] 部分成功。
- [ ] 无权限。
- [ ] 离线。
- [ ] 数据过期。
- [ ] 取消。
- [ ] 重试。
- [ ] 并发冲突。

验收：交付不再只有正常页面，而是流程、状态、异常、恢复、反馈和下一步。

---

## 五、交互模式库

### 导航

Sidebar、Top Navigation、Tabs、Breadcrumb、Command Palette、Stepper。

### 输入

Form、Inline Edit、Auto Complete、Tag Input、Date Picker、Upload、Rich Text Editor。

### 数据

Table、List、Card、Tree、Timeline、Dashboard、Chart。

### 反馈

Toast、Alert、Inline Error、Progress、Skeleton、Empty State、Result Page。

### 操作

Create、Edit、Delete、Batch、Drag、Undo、Retry、Save Draft。

### 协作

Comment、Mention、Share、Notification、Presence、Version History、Conflict Resolution。

每个模式记录：用户目标、适用/不适用场景、流程、状态、原则、错误案例、优秀产品、工程实现、可访问性和指标。

验收：面对需求能说明它属于哪种模式、有哪些方案、为什么选当前方案、主要风险是什么。

## 六、复杂 B 端交互

### 表格

- [ ] 信息层级、默认列和列配置。
- [ ] 固定列、横向滚动、排序、筛选和分页。
- [ ] 批量操作、跨页选择和不可选行。
- [ ] 行内编辑、树形数据和保存视图。
- [ ] 导出、权限字段和大数据性能。

### 表单

- [ ] 字段分组、默认值、必填与选填。
- [ ] 校验时机、异步校验和字段依赖。
- [ ] 草稿、自动保存、离开提醒。
- [ ] 提交失败保留输入。
- [ ] 服务端错误和多人冲突。

### 权限

- [ ] 无权限、只读、字段权限、数据范围。
- [ ] 临时权限、申请权限和权限过期。
- [ ] 不只隐藏功能，还说明原因和获取方式。

### 审批

- [ ] 当前节点、下一节点、审批人和预计时间。
- [ ] 驳回、撤回、转交、加签、抄送。
- [ ] 历史、超时、异常中断和并行审批。

---

## 七、视觉基础、动效与无障碍

学习视觉层级、对齐、亲密性、重复、对比、留白、字体、色彩角色、图标和动效目的。

Todo：

- [ ] 页面只有一个主要视觉焦点。
- [ ] 使用系统化间距和有限字体层级。
- [ ] 颜色不是唯一信息表达方式。
- [ ] 动效说明元素从哪来、去哪和状态如何变化。
- [ ] 高频操作避免冗长动画。
- [ ] 支持键盘、焦点和减少动画设置。

验收：能独立完成结构清晰、层级合理、可访问的中保真方案。

---

## 八、AI 产品交互

### Chat 与 Streaming

- [ ] 澄清需求、多轮、停止、重试、继续。
- [ ] 首 Token、部分结果、断线、中断和恢复。

### Artifact

- [ ] 创建、局部修改、版本、Diff、恢复、导出。
- [ ] 区分用户修改和 AI 修改。

### Tool Call 与 Approval

- [ ] 展示工具、参数、影响、风险和结果。
- [ ] 高风险写操作必须确认。

### Agent Task

- [ ] 目标、计划、步骤、当前进度。
- [ ] 暂停、取消、恢复、失败步骤和人工接管。

### Citation 与 Memory

- [ ] 引用定位原文、来源日期和推断说明。
- [ ] 记忆可查看、修改、删除、关闭并说明作用范围。

验收：用户始终知道 AI 正在做什么，可以停止、修改、确认、恢复，并能判断结果可信度。
