# Command Palette 命令面板

命令面板是临时打开的搜索与执行界面，用于从大量命令或资源中快速选择。它提高熟练用户效率，但不能成为新手发现功能的唯一入口。

## 能力边界与前置知识

本文假定已经能区分站点地图、用户流程、页面状态与服务端授权。完成后应能从真实任务选择Command Palette 命令面板，实现它专属的路由、状态和键盘行为，并通过深链、窄屏和辅助技术验证。

不把视觉上相似的控件自动归为Command Palette 命令面板。模式成立需要信息关系、操作模型和程序化语义同时匹配。

## 选择条件

| 方案 | 适用条件 | 收益 | 主要风险 |
| --- | --- | --- | --- |
| 只搜命令 | 编辑器与开发工具 | 语义一致 | 资源跳转另有入口 |
| 命令 + 资源 | 统一快速入口 | 效率高 | 同名结果与作用域复杂 |
| 分步参数面板 | 命令需要少量参数 | 保持键盘流 | 状态与返回更复杂 |
| 普通搜索页 | 结果需详细比较 | 可深链和浏览 | 执行路径更长 |

## 信息与交互模型

Command Palette 命令面板至少维护稳定项 ID、可见名称、目标或受控内容、当前状态、权限条件和来源范围。显示顺序可以变化，但稳定 ID 不随改名或本地化改变。

### 专属机制

- 查询输入通常采用 combobox，DOM 焦点保持在输入框。
- 活动结果通过 aria-activedescendant 或等价完整模式表达。
- 命令 ID、显示名称、快捷键、作用域和启用条件分别存储。
- 输入法组合结束前不执行搜索提交或快捷命令。
- 远程结果用取消或序列号防止乱序响应覆盖新查询。
- 禁用命令显示缺失上下文，不从结果中无解释消失。
- 导航、只读命令和写命令分区，危险命令不因排序直接执行。
- 关闭后焦点回到触发点；执行跳转后焦点进入新页合理起点。

```json
{
  "dialogId": "command-palette",
  "query": "rename",
  "scope": {
    "kind": "editor",
    "resourceId": "file-42"
  },
  "activeOptionId": "command-rename-symbol",
  "results": [
    {
      "commandId": "rename-symbol",
      "label": "重命名符号",
      "enabled": true,
      "risk": "reversible"
    }
  ]
}
```

命令面板通常是有名称的 dialog，内部查询框采用 combobox，结果采用 listbox 或按实际内容选择其他弹出结构。DOM 焦点保留在输入框，活动结果用 aria-activedescendant 表达。命令 ID、显示名称、作用域、启用条件和风险级别分开；执行时重新读取当前资源与权限。

### 状态

| 状态 | 可见结果 | 用户操作或系统转换 |
| --- | --- | --- |
| 关闭 | 触发器可见 | 快捷键或按钮打开 |
| 查询中 | 输入保有焦点 | 取消旧查询 |
| 有结果 | 一个活动候选 | 方向键与 Enter |
| 无结果 | 说明作用域与替代方式 | 修改查询 |
| 命令禁用 | 显示原因 | 补足上下文或权限 |
| 执行失败 | 保留查询和命令 | 修正、重试或关闭 |

命令面板分别维护打开状态、查询文本、活动结果 ID、候选命令能力和执行状态。活动结果只表示 Enter 将指向谁，不表示命令已执行；关闭面板也不能把正在服务端执行的写操作当作取消。

## 路由、历史与持久化

- Command Palette 命令面板中的导航目的地使用真实链接并支持新标签、复制和后退。
- 同页选择是否写入 URL，依据分享和刷新恢复需要决定。
- 临时面板关闭后恢复触发点或进入新页面的合理起点。
- 旧 URL 先完成规范化重定向，再计算当前状态。
- 用户偏好与业务当前状态分开保存；权限和路由永远重新计算。
- 会话过期后只保存安全目标路径，重新认证后重新授权。

## 键盘与无障碍

先使用与模式匹配的原生 HTML。只有Command Palette 命令面板确实属于 ARIA 复合控件时才接管方向键。

验证：

- Command Palette 命令面板的可见标签进入可访问名称；
- 页面导航使用链接而不是模拟点击区域；
- DOM 顺序与视觉顺序一致；
- 焦点指示清晰且不被固定层遮挡；
- 当前页、选中项、展开状态和禁用状态分别表达；
- 异步结果更新不无故抢焦点；
- 关闭临时层后焦点位置可预测；
- 320 CSS px 等效宽度和 200% 缩放下仍可操作；
- 不依赖 hover、颜色、图标或复杂手势完成任务。

### 名称、角色、状态和值

- Command Palette 命令面板每个可操作项都有不依赖图标的名称。
- 同一页面存在多个 navigation 地标时按用途分别命名。
- 链接、按钮、tab 和 option 不因视觉相似而混用角色。
- 输入框通过 `aria-activedescendant` 指向当前候选，DOM 焦点不移入结果列表；候选的禁用原因进入可访问描述，页面当前位置和命令活动项不使用同一状态属性。
- aria-expanded 只放在真实控制可折叠内容的元素上。
- aria-controls 引用的目标 ID 在当前 DOM 中存在且唯一。
- 禁用原因对键盘和读屏用户可获得，不只改变透明度。
- 动态数量或结果更新使用适合频率的状态消息，不反复打断当前阅读。

### 焦点验收

1. 从浏览器地址栏开始，只用 Tab 和Command Palette 命令面板规定按键完成主任务。
2. 打开临时界面，确认焦点进入可预测起点。
3. 在异步请求返回前切换目标，确认焦点不被旧结果夺走。
4. 关闭、取消或按 Escape，确认焦点回到触发点或新页面逻辑起点。
5. 收起包含焦点的区域，确认焦点先移到仍可见的控制项。
6. 触发权限变化，确认焦点不会落在被删除入口之后的未知位置。
7. 放大并滚动，确认固定层不完全遮挡焦点。
8. 使用屏幕阅读器的地标、链接或控件列表确认名称与状态。

### 发布前检查

- Command Palette 命令面板的可见名称、程序化名称和产品术语表一致。
- 默认项不依赖客户端旧缓存，能从当前 URL 或数据重建。
- 权限改变后入口、目标页面和搜索结果使用同一能力口径。
- 快速连续操作不会让旧响应覆盖新上下文。
- 浏览器后退和前进不会重复执行写命令。
- 本地化长文本、从右到左语言和系统大字体有明确布局结果。
- 关闭 JavaScript 后，属于页面导航的链接仍能到达目标。
- 埋点不保存原始搜索词、对象名称或其他敏感文本。

## 案例 1：代码编辑器搜索带上下文的重构命令

### 约束与输入

- 命令含 ID、名称、快捷键、when 条件与风险级别。
- 当前编辑器可能没有选择文本。

### 设计与实现

1. 快捷键与可见菜单项都能打开。
2. 输入框采用 combobox，结果列表显示禁用原因。
3. 模糊匹配只排序，不改变命令 ID。
4. 重构命令执行前读取当前文档版本和选择。
5. 执行失败回到命令结果并保留查询。

### 验证结果

- 方向键移动活动候选，DOM 焦点留在输入。
- 无选择时重构命令禁用且可解释。
- 同名命令按扩展来源区分。

### 失败分支

文档切换后旧命令仍启用并作用于错误文件。修正为执行时重新读取活动编辑器上下文。


## 案例 2：运营后台快速跳转客户并执行只读动作

### 约束与输入

- 可搜索客户与“查看账单”等只读命令；客户名称可能重复。
- 结果按租户和权限过滤。

### 设计与实现

1. 查询结果区分客户资源和命令类别。
2. 客户候选显示稳定辅助信息，不显示敏感字段。
3. 选择客户先进入详情，不直接执行写操作。
4. 远程查询使用序列号取消乱序响应。
5. 日志记录命令类别，不记录原始客户搜索词。

### 验证结果

- 同名客户可正确区分。
- 无权限客户不出现在候选和计数。
- 窄屏和读屏能获知结果数量与类别。

### 失败分支

模糊搜索把“删除客户”排在“查看客户”前。修正为写命令独立分区，并要求进入目标页复核，不从面板直接执行。


## 工程实现

### 数据来源

命令注册表提供稳定命令 ID、参数 Schema、适用范围和风险等级；当前对象与服务端能力决定候选是否出现或禁用。执行时重新读取作用对象和权限，搜索索引中的旧能力结果不能授权命令。

### 更新顺序

1. 解析Command Palette 命令面板最终规范化 URL 或当前控件上下文。
2. 获取当前主体、范围和能力。
3. 过滤不可发现项并删除无意义空组。
4. 计算当前、选中、展开和禁用状态。
5. 渲染稳定 DOM 顺序和可访问关系。
6. 异步内容返回时确认请求仍属于当前项。
7. 路由或数据变化后重新计算，不复用过期布尔状态。

### 不变量

- Command Palette 命令面板的稳定项 ID 唯一；
- 当前页或选中项数量符合模式定义；
- 程序化状态与可见状态一致；
- 隐藏内容中没有可到达焦点；
- 受控对象和控制项引用均存在；
- 失效异步响应不更新新上下文；
- 权限变化后旧入口不能继续读取数据；
- 浏览器后退恢复页面而不重复写操作。

## 窄屏与长内容

窄屏方案必须重新定义布局，不只是隐藏内容：

- Command Palette 命令面板在窄屏使用触发器时，触发器有清晰名称和展开状态；
- 打开后阅读顺序从模式标题到当前项再到其他项；
- 关闭方式包括明确按钮，并在适用时支持 Escape；
- 发生页面跳转后不错误恢复到旧触发器；
- 长标签允许换行或进入更适合的布局；
- 滚动容器不会让当前项和焦点长期不可见；
- 横竖屏切换不把桌面偏好错误映射为移动端打开状态。

## 调试与观测

### 外向内调试

1. 固定Command Palette 命令面板的角色、URL、视口、语言、数据和功能开关。
2. 检查页面标题、当前项、可访问名称和 DOM 焦点。
3. 检查路由匹配、重定向、请求取消和乱序响应。
4. 检查权限能力与目标页面授权结果。
5. 从深链、刷新、后退和新标签页重复任务。
6. 用键盘、读屏、窄屏和长文本回归。

### 指标

- Command Palette 命令面板固定任务到达正确目标的成功率；
- 第一次选择正确率；
- 选择后短时间返回或改用搜索的比例；
- 切换响应延迟和失败率；
- 深链恢复失败率；
- 键盘阻断、焦点丢失与状态未播报问题；
- 按角色、入口、平台和新旧用户分群的任务时间。

点击数不能独立证明导航改善。点击增加可能代表更常用，也可能代表用户绕路。

## 失败注入

- Command Palette 命令面板当前目标在操作期间被删除；
- 角色权限被撤销；
- 请求延迟并乱序返回；
- 当前项名称增长到 40 个汉字；
- 页面在 320 CSS px 与 200% 缩放下打开；
- 会话过期后从深链恢复；
- 旧 URL 发生重定向；
- 用户用浏览器后退恢复缓存页面；
- 当前项在不可见或收起区域；
- JavaScript 加载失败，检查基础链接是否仍可用。

## 综合练习

为一个真实管理工具实现Command Palette 命令面板。交付信息模型、状态表、两个不同约束的可运行原型、路由与权限契约、键盘记录、窄屏结果和失败注入报告。

验收：

- 能解释为什么选Command Palette 命令面板而不是表中其他方案；
- 深链、刷新、后退和新标签页行为符合任务；
- 当前、选中、焦点、展开和禁用没有混用；
- 异步失败保留可用上下文；
- 目标页面执行独立服务端授权；
- 键盘和屏幕阅读器完成主任务；
- 窄屏、长文本和放大不隐藏唯一操作；
- 两个案例都能从界面结果对账到路由或权威数据。
