# 交互设计模块学习笔记

每完成一个原则、模式或专项模块，就在这里形成可检索的设计知识。复制 [交互笔记模板](../notes-template.md)。日常单个任务分析放在 `daily/`，跨案例总结出的稳定规律放在这里。

建议结构：

```text
notes/
├── principles/
├── information-architecture/
├── flows-states/
├── interaction-patterns/
├── b2b-complexity/
├── visual-motion-accessibility/
└── ai-ux/
```

笔记应包含用户目标、原则、适用边界、正反案例、状态与无障碍要求、工程约束以及验证方式。
