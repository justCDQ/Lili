# Tabs 标签页

Tabs 在同一上下文中显示一组互斥内容面板。它是复合控件，不是把独立站点页面排成一行的通用导航。

## 能力边界与前置知识

本文假定已经能区分站点地图、用户流程、页面状态与服务端授权。完成后应能从真实任务选择Tabs 标签页，实现它专属的路由、状态和键盘行为，并通过深链、窄屏和辅助技术验证。

不把视觉上相似的控件自动归为Tabs 标签页。模式成立需要信息关系、操作模型和程序化语义同时匹配。

## 选择条件

| 方案 | 适用条件 | 收益 | 主要风险 |
| --- | --- | --- | --- |
| 自动激活 Tabs | 面板已预加载且切换无延迟 | 方向键浏览快 | 延迟会让焦点移动变等待 |
| 手动激活 Tabs | 面板按需加载 | 用户决定何时切换 | 多一次 Enter/Space |
| 普通页面链接 | 内容有独立 URL 和页面生命周期 | 深链、历史自然 | 切换不是复合控件 |
| 同页顺序内容 | 用户需要同时比较 | 内容可一起阅读 | 页面更长 |

## 信息与交互模型

Tabs 标签页至少维护稳定项 ID、可见名称、目标或受控内容、当前状态、权限条件和来源范围。显示顺序可以变化，但稳定 ID 不随改名或本地化改变。

### 专属机制

- tablist 给整组标签命名；每个 tab 与一个 tabpanel 双向关联。
- 选中状态、DOM 焦点和面板可见性是三个需要同步的状态。
- 自动激活只适合面板预加载且切换无可感知延迟。
- 手动激活时方向键只移动焦点，Enter 或 Space 才改变选中面板。
- 一个 tab 保留 tabindex=0，其他标签为 -1，形成 roving tabindex。
- 动态删除标签后把焦点放到相邻标签并激活对应面板。
- 面板加载失败保留 tab 与重试，不用空面板掩盖错误。
- 需要分享和刷新恢复时把选中 ID 写入 URL，而不是数组序号。

```json
{
  "tablistId": "ticket-sections",
  "activation": "manual",
  "tabs": [
    {
      "id": "activity-tab",
      "panelId": "activity-panel",
      "selected": true,
      "tabIndex": 0
    },
    {
      "id": "attachments-tab",
      "panelId": "attachments-panel",
      "selected": false,
      "tabIndex": -1
    }
  ]
}
```

tablist 为整组标签提供名称；每个 tab 用 aria-controls 指向唯一 tabpanel，tabpanel 用 aria-labelledby 回指标签。选中项使用 aria-selected=true；roving tabindex 使一个 tab 留在页面 Tab 序列。自动激活只有在面板预加载且切换无可感知延迟时成立，否则方向键移动焦点、Enter 或 Space 激活。

### 状态

| 状态 | 可见结果 | 用户操作或系统转换 |
| --- | --- | --- |
| 未激活 | aria-selected=false，tabindex=-1 | 方向键可移动焦点 |
| 已激活 | aria-selected=true，tabindex=0 | 关联面板可见 |
| 焦点未激活 | 手动激活模式下焦点与选中分离 | Enter/Space 激活 |
| 加载 | 标签保持可用，面板表达 busy | 取消旧请求 |
| 失败 | 面板就地显示错误与重试 | 不改成空面板 |
| 被删除 | 动态标签移除 | 焦点移动到相邻标签 |

每个状态都要定义进入和退出条件。视觉高亮、DOM 焦点、当前页面、选中面板和已执行命令是不同概念，不能用一个 `active` 布尔值同时表示。

## 路由、历史与持久化

- Tabs 标签页中的导航目的地使用真实链接并支持新标签、复制和后退。
- 同页选择是否写入 URL，依据分享和刷新恢复需要决定。
- 临时面板关闭后恢复触发点或进入新页面的合理起点。
- 旧 URL 先完成规范化重定向，再计算当前状态。
- 用户偏好与业务当前状态分开保存；权限和路由永远重新计算。
- 会话过期后只保存安全目标路径，重新认证后重新授权。

## 键盘与无障碍

先使用与模式匹配的原生 HTML。只有Tabs 标签页确实属于 ARIA 复合控件时才接管方向键。

验证：

- Tabs 标签页的可见标签进入可访问名称；
- 页面导航使用链接而不是模拟点击区域；
- DOM 顺序与视觉顺序一致；
- 焦点指示清晰且不被固定层遮挡；
- 当前页、选中项、展开状态和禁用状态分别表达；
- 异步结果更新不无故抢焦点；
- 关闭临时层后焦点位置可预测；
- 320 CSS px 等效宽度和 200% 缩放下仍可操作；
- 不依赖 hover、颜色、图标或复杂手势完成任务。

### 名称、角色、状态和值

- Tabs 标签页每个可操作项都有不依赖图标的名称。
- 同一页面存在多个 navigation 地标时按用途分别命名。
- 链接、按钮、tab 和 option 不因视觉相似而混用角色。
- aria-current 表达当前页面或步骤；aria-selected 只用于支持选择语义的控件。
- aria-expanded 只放在真实控制可折叠内容的元素上。
- aria-controls 引用的目标 ID 在当前 DOM 中存在且唯一。
- 禁用原因对键盘和读屏用户可获得，不只改变透明度。
- 动态数量或结果更新使用适合频率的状态消息，不反复打断当前阅读。

### 焦点验收

1. 从浏览器地址栏开始，只用 Tab 和Tabs 标签页规定按键完成主任务。
2. 打开临时界面，确认焦点进入可预测起点。
3. 在异步请求返回前切换目标，确认焦点不被旧结果夺走。
4. 关闭、取消或按 Escape，确认焦点回到触发点或新页面逻辑起点。
5. 收起包含焦点的区域，确认焦点先移到仍可见的控制项。
6. 触发权限变化，确认焦点不会落在被删除入口之后的未知位置。
7. 放大并滚动，确认固定层不完全遮挡焦点。
8. 使用屏幕阅读器的地标、链接或控件列表确认名称与状态。

### 发布前检查

- Tabs 标签页的可见名称、程序化名称和产品术语表一致。
- 默认项不依赖客户端旧缓存，能从当前 URL 或数据重建。
- 权限改变后入口、目标页面和搜索结果使用同一能力口径。
- 快速连续操作不会让旧响应覆盖新上下文。
- 浏览器后退和前进不会重复执行写命令。
- 本地化长文本、从右到左语言和系统大字体有明确布局结果。
- 关闭 JavaScript 后，属于页面导航的链接仍能到达目标。
- 埋点不保存原始搜索词、对象名称或其他敏感文本。

## 案例 1：工单详情切换活动、附件和审计记录

### 约束与输入

- 三个面板共享同一工单 ID；附件按需加载；审计只读。
- 面板选择写入 URL fragment 以支持分享。

### 设计与实现

1. 使用 tablist、tab、tabpanel 的完整关联。
2. 采用手动激活，避免方向键经过“附件”就触发网络等待。
3. 激活时取消上一个尚未完成请求。
4. URL fragment 改变后同步选中项，但不重新创建工单上下文。
5. 失败只替换当前面板内容。

### 验证结果

- 左右方向键移动标签焦点；Enter 激活。
- 刷新 fragment 恢复同一面板。
- 加载失败时其他面板仍可用。

### 失败分支

附件响应晚于审计响应并覆盖当前面板。修正为请求绑定 panel ID，失效响应不得更新已切换面板。


## 案例 2：分析页面切换日、周、月视图

### 约束与输入

- 数据已预取；三个面板结构相同；用户高频比较。
- 窄屏下标签名称仍可完整显示。

### 设计与实现

1. 使用自动激活，因为切换无可感知延迟。
2. 方向键移动后立即更新图表、标题和数据表。
3. 保持一个 tab 在 Tab 序列中。
4. 图表和替代数据表共享同一时间粒度。
5. 窄屏允许标签横向滚动，但焦点项自动进入可视区。

### 验证结果

- 方向键快速遍历无延迟和请求风暴。
- 读屏获知选中项与面板名称。
- 日/周/月数据口径不会混在同一缓存键。

### 失败分支

自动激活后来改为远程请求，方向键每次移动都等待。应改为手动激活或恢复预取，不保留失效的自动策略。


## 工程实现

### 数据来源

导航结构来自信息架构与路由配置；权限能力来自服务端；徽标和动态状态可以独立请求。目标页面必须再次授权，不能信任模式是否显示。

### 更新顺序

1. 解析Tabs 标签页最终规范化 URL 或当前控件上下文。
2. 获取当前主体、范围和能力。
3. 过滤不可发现项并删除无意义空组。
4. 计算当前、选中、展开和禁用状态。
5. 渲染稳定 DOM 顺序和可访问关系。
6. 异步内容返回时确认请求仍属于当前项。
7. 路由或数据变化后重新计算，不复用过期布尔状态。

### 不变量

- Tabs 标签页的稳定项 ID 唯一；
- 当前页或选中项数量符合模式定义；
- 程序化状态与可见状态一致；
- 隐藏内容中没有可到达焦点；
- 受控对象和控制项引用均存在；
- 失效异步响应不更新新上下文；
- 权限变化后旧入口不能继续读取数据；
- 浏览器后退恢复页面而不重复写操作。

## 窄屏与长内容

窄屏方案必须重新定义布局，不只是隐藏内容：

- Tabs 标签页在窄屏使用触发器时，触发器有清晰名称和展开状态；
- 打开后阅读顺序从模式标题到当前项再到其他项；
- 关闭方式包括明确按钮，并在适用时支持 Escape；
- 发生页面跳转后不错误恢复到旧触发器；
- 长标签允许换行或进入更适合的布局；
- 滚动容器不会让当前项和焦点长期不可见；
- 横竖屏切换不把桌面偏好错误映射为移动端打开状态。

## 调试与观测

### 外向内调试

1. 固定Tabs 标签页的角色、URL、视口、语言、数据和功能开关。
2. 检查页面标题、当前项、可访问名称和 DOM 焦点。
3. 检查路由匹配、重定向、请求取消和乱序响应。
4. 检查权限能力与目标页面授权结果。
5. 从深链、刷新、后退和新标签页重复任务。
6. 用键盘、读屏、窄屏和长文本回归。

### 指标

- Tabs 标签页固定任务到达正确目标的成功率；
- 第一次选择正确率；
- 选择后短时间返回或改用搜索的比例；
- 切换响应延迟和失败率；
- 深链恢复失败率；
- 键盘阻断、焦点丢失与状态未播报问题；
- 按角色、入口、平台和新旧用户分群的任务时间。

点击数不能独立证明导航改善。点击增加可能代表更常用，也可能代表用户绕路。

## 失败注入

- Tabs 标签页当前目标在操作期间被删除；
- 角色权限被撤销；
- 请求延迟并乱序返回；
- 当前项名称增长到 40 个汉字；
- 页面在 320 CSS px 与 200% 缩放下打开；
- 会话过期后从深链恢复；
- 旧 URL 发生重定向；
- 用户用浏览器后退恢复缓存页面；
- 当前项在不可见或收起区域；
- JavaScript 加载失败，检查基础链接是否仍可用。

## 综合练习

为一个真实管理工具实现Tabs 标签页。交付信息模型、状态表、两个不同约束的可运行原型、路由与权限契约、键盘记录、窄屏结果和失败注入报告。

验收：

- 能解释为什么选Tabs 标签页而不是表中其他方案；
- 深链、刷新、后退和新标签页行为符合任务；
- 当前、选中、焦点、展开和禁用没有混用；
- 异步失败保留可用上下文；
- 目标页面执行独立服务端授权；
- 键盘和屏幕阅读器完成主任务；
- 窄屏、长文本和放大不隐藏唯一操作；
- 两个案例都能从界面结果对账到路由或权威数据。
