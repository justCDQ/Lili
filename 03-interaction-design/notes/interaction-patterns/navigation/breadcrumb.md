# Breadcrumb 面包屑

面包屑按稳定层级显示当前位置的祖先路径，帮助定向和向上导航。它表达信息架构，不表达用户刚才访问过的历史。

## 能力边界与前置知识

本文假定已经能区分站点地图、用户流程、页面状态与服务端授权。完成后应能从真实任务选择Breadcrumb 面包屑，实现它专属的路由、状态和键盘行为，并通过深链、窄屏和辅助技术验证。

不把视觉上相似的控件自动归为Breadcrumb 面包屑。模式成立需要信息关系、操作模型和程序化语义同时匹配。

## 选择条件

| 方案 | 适用条件 | 收益 | 主要风险 |
| --- | --- | --- | --- |
| 含当前页 | 需要完整位置说明 | 路径完整 | 与 h1 可能重复 |
| 只含祖先 | 当前页标题已清楚 | 更简洁 | 需要 aria-label 说明区域 |
| 不使用 | 首页或浅层结构 | 避免冗余 | 失去深层定向 |
| 层级选择器 | 对象有多个可切换父级 | 可改变上下文 | 已不是普通面包屑 |

## 信息与交互模型

Breadcrumb 面包屑至少维护稳定项 ID、可见名称、目标或受控内容、当前状态、权限条件和来源范围。显示顺序可以变化，但稳定 ID 不随改名或本地化改变。

### 专属机制

- 用 navigation 地标和有序列表表达从根到当前位置的顺序。
- 祖先是稳定信息层级，不随搜索、通知或书签入口改变。
- 当前页可以显示为非链接，并用 aria-current=page 表达。
- 分隔符是装饰内容，不进入链接名称或可访问树。
- 对象多归属时选择产品定义的主要父级，其他关系用交叉链接。
- 对象移动后更新祖先路径，但稳定对象 URL 可以不变。
- 视觉折叠中间层时仍要保留可理解的祖先信息和键盘入口。
- 无权限祖先按策略隐藏、概括或进入受控说明，不能泄露名称。

```json
{
  "landmarkLabel": "面包屑",
  "items": [
    {
      "id": "docs",
      "label": "文档",
      "href": "/docs"
    },
    {
      "id": "api",
      "label": "API",
      "href": "/docs/api"
    },
    {
      "id": "upload",
      "label": "上传对象",
      "current": true
    }
  ]
}
```

面包屑使用有名称的 navigation 地标与有序列表。祖先使用普通链接；当前页可用非链接文本并设置 aria-current=page。分隔符是装饰，不进入可访问名称。路径来自稳定信息架构，不读取 referrer 或浏览历史。

### 状态

| 状态 | 可见结果 | 用户操作或系统转换 |
| --- | --- | --- |
| 浅层 | 不显示或只显示一个祖先 | 避免无意义路径 |
| 深层 | 有序列出祖先 | 当前页用 aria-current |
| 对象移动 | 使用新主要父级 | 旧 URL 重定向 |
| 祖先无权限 | 跳过或进入受控说明 | 不泄露受限名称 |
| 长路径 | 视觉折叠中间项 | 可访问路径仍可理解 |
| 失效 | 目标已删除 | 进入明确失效状态 |

面包屑状态由当前资源 ID、规范父链版本、可见祖先和当前路由共同计算。资源移动后旧父链属于过期数据，不是另一个“active”项；当前页也始终只在路径末端表达。

## 路由、历史与持久化

- Breadcrumb 面包屑中的导航目的地使用真实链接并支持新标签、复制和后退。
- 同页选择是否写入 URL，依据分享和刷新恢复需要决定。
- 临时面板关闭后恢复触发点或进入新页面的合理起点。
- 旧 URL 先完成规范化重定向，再计算当前状态。
- 用户偏好与业务当前状态分开保存；权限和路由永远重新计算。
- 会话过期后只保存安全目标路径，重新认证后重新授权。

## 键盘与无障碍

先使用与模式匹配的原生 HTML。只有Breadcrumb 面包屑确实属于 ARIA 复合控件时才接管方向键。

验证：

- Breadcrumb 面包屑的可见标签进入可访问名称；
- 页面导航使用链接而不是模拟点击区域；
- DOM 顺序与视觉顺序一致；
- 焦点指示清晰且不被固定层遮挡；
- 当前页、选中项、展开状态和禁用状态分别表达；
- 异步结果更新不无故抢焦点；
- 关闭临时层后焦点位置可预测；
- 320 CSS px 等效宽度和 200% 缩放下仍可操作；
- 不依赖 hover、颜色、图标或复杂手势完成任务。

### 名称、角色、状态和值

- Breadcrumb 面包屑每个可操作项都有不依赖图标的名称。
- 同一页面存在多个 navigation 地标时按用途分别命名。
- 链接、按钮、tab 和 option 不因视觉相似而混用角色。
- 路径末项使用 `aria-current="page"` 或不可点击文本，祖先保持普通链接；面包屑没有选择模型，不应出现 `aria-selected`、roving tabindex 或方向键接管。
- aria-expanded 只放在真实控制可折叠内容的元素上。
- aria-controls 引用的目标 ID 在当前 DOM 中存在且唯一。
- 禁用原因对键盘和读屏用户可获得，不只改变透明度。
- 动态数量或结果更新使用适合频率的状态消息，不反复打断当前阅读。

### 焦点验收

1. 从浏览器地址栏开始，只用 Tab 和Breadcrumb 面包屑规定按键完成主任务。
2. 打开临时界面，确认焦点进入可预测起点。
3. 在异步请求返回前切换目标，确认焦点不被旧结果夺走。
4. 关闭、取消或按 Escape，确认焦点回到触发点或新页面逻辑起点。
5. 收起包含焦点的区域，确认焦点先移到仍可见的控制项。
6. 触发权限变化，确认焦点不会落在被删除入口之后的未知位置。
7. 放大并滚动，确认固定层不完全遮挡焦点。
8. 使用屏幕阅读器的地标、链接或控件列表确认名称与状态。

### 发布前检查

- Breadcrumb 面包屑的可见名称、程序化名称和产品术语表一致。
- 默认项不依赖客户端旧缓存，能从当前 URL 或数据重建。
- 权限改变后入口、目标页面和搜索结果使用同一能力口径。
- 快速连续操作不会让旧响应覆盖新上下文。
- 浏览器后退和前进不会重复执行写命令。
- 本地化长文本、从右到左语言和系统大字体有明确布局结果。
- 关闭 JavaScript 后，属于页面导航的链接仍能到达目标。
- 埋点不保存原始搜索词、对象名称或其他敏感文本。

## 案例 1：文档站从 API 方法返回所属模块

### 约束与输入

- 路径：文档/API/存储/上传对象；页面可从搜索进入。
- 模块名有长语言版本。

### 设计与实现

1. 从文档结构生成主要父级，不读取 referrer。
2. 使用 nav + ol + 链接。
3. 当前页使用 aria-current=page，分隔符隐藏于可访问树。
4. 搜索进入仍显示相同路径。
5. 窄屏截短中间视觉文本时保留完整可访问名称。

### 验证结果

- 搜索、书签和站内点击得到同一路径。
- 祖先链接均可刷新和分享。
- 结构化数据与可见路径一致。

### 失败分支

旧实现把搜索结果页放进面包屑，导致同一方法因入口不同出现不同父级。修正为只使用稳定文档层级。


## 案例 2：资产树从设备详情返回区域和机房

### 约束与输入

- 设备可移动机房；当前用户可能无权查看区域详情。
- 旧书签仍指向设备稳定 URL。

### 设计与实现

1. 设备详情读取当前主要位置版本。
2. 按权限决定祖先是否可链接和显示。
3. 设备移动后更新路径，旧设备 URL 不变。
4. 无权查看机房时保留可理解的受限上级或安全范围。
5. 审计记录移动前后位置。

### 验证结果

- 设备 URL 稳定；路径反映当前归属。
- 无权限不暴露机房敏感名称。
- 移动并发不会出现混合新旧祖先。

### 失败分支

设备位置缓存过期，面包屑显示旧机房但操作作用于新机房。修正为路径与详情使用同一版本快照并表达过期。


## 工程实现

### 数据来源

祖先链由当前资源的规范父关系生成，并与详情数据使用同一版本快照。服务端在返回路径前过滤不可发现祖先；点击任一祖先后，目标资源仍按当前权限读取，不能把“曾出现在路径中”视为访问授权。

### 更新顺序

1. 解析Breadcrumb 面包屑最终规范化 URL 或当前控件上下文。
2. 获取当前主体、范围和能力。
3. 过滤不可发现项并删除无意义空组。
4. 计算当前、选中、展开和禁用状态。
5. 渲染稳定 DOM 顺序和可访问关系。
6. 异步内容返回时确认请求仍属于当前项。
7. 路由或数据变化后重新计算，不复用过期布尔状态。

### 不变量

- Breadcrumb 面包屑的稳定项 ID 唯一；
- 当前页或选中项数量符合模式定义；
- 程序化状态与可见状态一致；
- 隐藏内容中没有可到达焦点；
- 受控对象和控制项引用均存在；
- 失效异步响应不更新新上下文；
- 权限变化后旧入口不能继续读取数据；
- 浏览器后退恢复页面而不重复写操作。

## 窄屏与长内容

窄屏方案必须重新定义布局，不只是隐藏内容：

- Breadcrumb 面包屑在窄屏使用触发器时，触发器有清晰名称和展开状态；
- 打开后阅读顺序从模式标题到当前项再到其他项；
- 关闭方式包括明确按钮，并在适用时支持 Escape；
- 发生页面跳转后不错误恢复到旧触发器；
- 长标签允许换行或进入更适合的布局；
- 滚动容器不会让当前项和焦点长期不可见；
- 横竖屏切换不把桌面偏好错误映射为移动端打开状态。

## 调试与观测

### 外向内调试

1. 固定Breadcrumb 面包屑的角色、URL、视口、语言、数据和功能开关。
2. 检查页面标题、当前项、可访问名称和 DOM 焦点。
3. 检查路由匹配、重定向、请求取消和乱序响应。
4. 检查权限能力与目标页面授权结果。
5. 从深链、刷新、后退和新标签页重复任务。
6. 用键盘、读屏、窄屏和长文本回归。

### 指标

- Breadcrumb 面包屑固定任务到达正确目标的成功率；
- 第一次选择正确率；
- 选择后短时间返回或改用搜索的比例；
- 切换响应延迟和失败率；
- 深链恢复失败率；
- 键盘阻断、焦点丢失与状态未播报问题；
- 按角色、入口、平台和新旧用户分群的任务时间。

点击数不能独立证明导航改善。点击增加可能代表更常用，也可能代表用户绕路。

## 失败注入

- Breadcrumb 面包屑当前目标在操作期间被删除；
- 角色权限被撤销；
- 请求延迟并乱序返回；
- 当前项名称增长到 40 个汉字；
- 页面在 320 CSS px 与 200% 缩放下打开；
- 会话过期后从深链恢复；
- 旧 URL 发生重定向；
- 用户用浏览器后退恢复缓存页面；
- 当前项在不可见或收起区域；
- JavaScript 加载失败，检查基础链接是否仍可用。

## 综合练习

为一个真实管理工具实现Breadcrumb 面包屑。交付信息模型、状态表、两个不同约束的可运行原型、路由与权限契约、键盘记录、窄屏结果和失败注入报告。

验收：

- 能解释为什么选Breadcrumb 面包屑而不是表中其他方案；
- 深链、刷新、后退和新标签页行为符合任务；
- 当前、选中、焦点、展开和禁用没有混用；
- 异步失败保留可用上下文；
- 目标页面执行独立服务端授权；
- 键盘和屏幕阅读器完成主任务；
- 窄屏、长文本和放大不隐藏唯一操作；
- 两个案例都能从界面结果对账到路由或权威数据。
