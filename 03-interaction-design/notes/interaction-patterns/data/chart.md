# Chart 图表

图表把定量关系编码为位置、长度、角度、颜色或形状。选择编码必须服从比较任务，并提供不依赖视觉的等价数据。

## 数据与任务边界

图表的输入不是一组待装饰的数字，而是明确的分析问题、数据类型和聚合结果。设计必须保留从原始值到比例尺、视觉编码和等价表格的转换链，才能检查轴、缺失值和统计口径。

前置知识：定量与分类变量、聚合与分组、坐标轴、SVG/Canvas 的差异、键盘焦点和表格语义。

## 数据模型

```json
{
  "chartId": "retention-cohorts",
  "type": "line",
  "x": {
    "field": "week",
    "unit": "week"
  },
  "y": {
    "field": "retention",
    "unit": "percent",
    "domain": [
      0,
      100
    ]
  },
  "series": [
    {
      "id": "cohort-2026-06",
      "label": "2026-06",
      "points": [
        [
          0,
          100
        ],
        [
          1,
          42
        ]
      ]
    }
  ]
}
```

`x`、`y` 同时定义字段和单位，比例尺的 domain 不能依靠绘图库从异常数据中任意推断。系列使用稳定 `id` 关联颜色、线型、图例和等价表格；`points` 中还应以显式状态表达缺失和低于检测限。

## 工作机制

- 先定义比较问题，再选择图形。
- 位置与长度通常比角度、面积更适合精确比较。
- 轴标题、单位、范围和基线明确。
- 缺失值、零与未采集使用不同表示。
- 颜色系列同时使用线型、标记或直接标签。
- 图旁提供文本结论与数据表。
- 交互 tooltip 不能承载唯一值。
- 样本量、分箱和聚合方法可检查。

```mermaid
flowchart LR
    D["原始数据与统计口径"] --> A["聚合、分箱与缺失处理"]
    A --> S["比例尺：domain → range"]
    S --> M["位置、长度、颜色或形状"]
    M --> G["可视图形"]
    A --> T["精确值表格"]
    G --> V["视觉判断"]
    T --> V
```


## 交互规则

- 键盘可到达系列或提供等价表格。
- 缩放和筛选有重置入口。
- 悬停信息也可通过焦点或固定详情取得。
- 隐藏系列后更新结论和可访问描述。
- 导出保留单位、时间窗和查询。
- 动画尊重 prefers-reduced-motion。

## Chart 图表状态

| 状态 | 专属行为 |
| --- | --- |
| 数据加载 | 轴和结论尚未形成，不显示虚假零值 |
| 可用 | 字段、单位、比例尺和样本量完整 |
| 缺失 | 点缺失，折线按规则断开 |
| 筛选 | 图、结论和等价表格使用同一查询 |
| 系列隐藏 | 更新图例、结论和表格 |
| 过期 | 保留旧图并显示更新时间 |

## 案例 1：产品团队查看留存分群曲线

### 约束与输入

- 4 个注册月 cohort；12 周；百分比与样本量。
- 需要比较第 1、4、8 周。

### 处理过程

1. 使用线图，x 为周、y 为 0–100% 留存。
2. 系列直接标注 cohort 名。
3. 数据表包含人数和百分比。
4. 缺失周显示为缺失，不连接为零。
5. 结论引用具体周和样本量。

### 失败分支

小 cohort 百分比波动被误读为趋势。修正为同时显示样本量并设置最小解释条件。

### 专属验证

- 分别抽取四个 cohort 的第 1、4、8 周，从人数复算留存率，图上点、提示框与表格误差不超过规定舍入精度。
- 某 cohort 第 8 周尚未到观察期时，折线在第 7 周结束，表格显示“未到观察期”，不得出现 0% 点。
- 隐藏颜色后仍能从末端直接标签和点形识别四条曲线；样本不足的 cohort 显示样本量警告。
- 切换 cohort 筛选后，结论文字、图形和等价表三者使用同一筛选集合。

## 案例 2：容量团队比较延迟分位数

### 约束与输入

- P50/P95/P99；三个版本；毫秒。
- 需要发现尾延迟回归。

### 处理过程

1. 使用共享 y 轴的折线或点图。
2. 不把分位数平均。
3. 版本发布时间作为注释事件。
4. 截断轴时明确范围并提供全范围视图。
5. 表格提供精确值。

### 失败分支

y 轴从 480ms 起始使 5ms 变化看似巨大。修正为依据任务展示合理基线并明确轴范围。

### 专属验证

- 版本发布点两侧各取一个时间桶，分别核对 P50、P95、P99，不能把三个分位数求平均。
- 将 y 轴下界设为 480 ms 时必须显示截断范围，并提供从零或全数据域开始的对照视图。
- 注入一个 8,000 ms 离群值后，P99 的变化可见，P50 不应被错误替换；表格保留原始毫秒单位。
- 键盘聚焦发布注释可取得版本号与发布时间，聚焦数据点可取得分位数、时间和精确值。

## 语义与键盘

- 图表区域先提供标题、时间窗、单位和一句由数据支持的结论。
- 数据表与图形使用同一份变换后数据，表头明确维度、系列和单位。
- 缩放、系列开关与重置均为可命名控件，状态变化不只体现在图形上。
- 支持逐点交互时，键盘焦点按数据顺序移动，提示信息与鼠标悬停一致。
- 系列不能只用颜色区分；直接标签、线型、点形或分面至少提供一种冗余编码。
- 图形刷新不抢走表格或控件焦点，高频数据变化只说明与当前分析有关的结果。

## Chart 图表工程实现

### 1. 比较绝对值优先位置或长度；角度和面积只用于粗略组成判断。

图表配置应先记录分析任务，再选择视觉编码：同一基线上的条形长度适合精确比较类别，折线位置适合观察连续时间趋势。饼图不适合大量类别或数值接近的精确排序，因为读者需要比较角度而非共享基线。

### 2. 比例尺保存 domain、range、scale type 与 clamp。对数轴不能接收零或负值；截断轴必须清晰显示。

比例尺配置应与数据快照一起进入测试和导出，避免同一数据在不同端得到不同坐标。输入超出 domain 时，`clamp` 决定是映射到边界还是继续外推；切换到对数轴前必须验证所有有效值大于零，并为无效点给出明确状态。

### 3. 缺失、零、低于检测限和不适用使用不同数据值与视觉。不要用零填补缺失折线。

数据层可用显式状态字段区分 `observed`、`missing`、`below-limit` 与 `not-applicable`，渲染层再决定断线、特殊标记或文字说明。把缺失转换成 `0` 会制造不存在的骤降，也会污染均值、总和和变化率。

### 4. 颜色编码同时提供直接标签、线型、点形或分面；检查高对比和色觉差异。

系列标识应在数据定义中稳定绑定颜色与第二编码，不能按当前排序临时分配颜色，否则刷新后同一系列会变色。测试时使用灰度、高对比模式和常见色觉差异模拟，确认读者仍能从标签、线型或点形识别系列。

### 5. SVG 元素不自动提供完整数据语义。为图表写文本结论和可访问数据表；交互点按需提供键盘详情。

图表容器应有可感知标题和简短结论，等价表格保留系列、时间、单位和精确值。若散点或柱形支持点击详情，应让同一目标可通过键盘到达，并在焦点变化时提供与鼠标提示框相同的信息。

### 6. 样本量、聚合、分箱、置信区间与异常值处理进入图表说明和导出。

图表数据契约需要携带聚合函数、分组维度、分箱边界、样本量和区间算法，不能只传最终坐标。导出结果也要包含这些元数据，使读者能够复算；异常值若被截尾或排除，说明中必须给出规则与受影响数量。

## Chart 图表调试

- 检查数据域与比例尺域，不从像素反推值。
- 注入零、负数、NaN、极大值和缺失点。
- 比较截断轴与完整轴对判断的影响。
- 关闭颜色，确认系列仍可区分。
- 键盘访问 tooltip 中的同一信息。
- 对账图中点、数据表和下载文件。

调试时保存聚合前样本、聚合结果、比例尺 domain/range、最终像素坐标和等价表格。随机抽取数据点从原始值复算坐标，再反向检查图形标记、提示框和表格是否指向同一记录。

## Chart 图表发布检查

- 图形选择匹配比较任务
- 轴、单位、基线与时间窗明确
- 缺失不被画成零
- 样本量与聚合可检查
- 颜色不是唯一系列编码
- 等价表格含完整关键值

失败注入包括 `0`、负数、`NaN`、缺失、单点序列、极端离群值、空分箱和截断轴。对数轴必须拒绝非正值或明确切换方案；窄屏下图形可滚动或重排，但单位、图例与等价表不可丢失。

## 综合练习

用同一留存数据制作折线图、文本结论和数据表；加入小样本、缺失周和筛选。再制作P50/P95/P99延迟图，证明未平均分位数。

验收用同一数据分别完成趋势判断和类别比较，证明图形选择与任务匹配；抽查所有轴端点、聚合值与样本量，关闭颜色后仍能识别系列，并仅用键盘取得任一交互数据点的精确值。

## 从比较任务选择视觉编码

| 比较任务 | 优先编码 | 谨慎使用 |
| --- | --- | --- |
| 精确比较数值 | 共同基线上的位置、长度 | 面积、角度 |
| 时间趋势 | 位置连接成线 | 平滑曲线隐藏真实点 |
| 排名 | 排序条形图、点图 | 饼图 |
| 部分与整体 | 堆叠条、100% 堆叠 | 多扇区饼图 |
| 分布 | 直方图、箱线图、点图 | 只显示平均值 |
| 两变量关系 | 散点图 | 双轴折线 |
| 空间关系 | 地图 | 用面积表示不相关数量 |

图表类型不是风格选择。先写出读者必须回答的问题，再确定字段、聚合和编码。

## 比例尺

比例尺把数据域映射到视觉范围：

```json
{
  "type": "linear",
  "domain": [0, 100],
  "range": [320, 0],
  "unit": "percent",
  "clamp": false
}
```

### 线性比例尺

相同数据差对应相同视觉距离。条形图通常从零基线开始，否则长度不再对应完整数量。

### 对数比例尺

适合跨多个数量级且值为正的数据。零和负数没有普通对数位置，不能静默丢弃。

### 时间比例尺

输入使用明确时间点或逻辑日期。跨夏令时的间隔不能简单按本地标签等距处理。

### 序数比例尺

类别没有自然数值距离。排序规则必须来自任务，而不是对象键的偶然顺序。

### 截断轴

折线图可以在明确场景截断轴以观察变化，但必须：

- 显示真实轴范围；
- 不让面积或长度被误读；
- 提供完整范围视图；
- 在结论中写出实际差值；
- 保留数据表。

## 缺失值

至少区分：

| 数据状态 | 数据字段 | 图形 |
| --- | --- | --- |
| 真实为零 | `0` | 零位置 |
| 未采集 | `null` + reason | 断点或缺失标记 |
| 不适用 | `null` + not-applicable | 明确文本 |
| 低于检测限 | 上限/下限信息 | 区间或专门标记 |
| 查询失败 | series error | 局部错误状态 |
| 仍在加载 | pending | 不绘制最终值 |

把缺失填成零会制造不存在的下降；直接连接折线会制造不存在的连续变化。

## 样本量与不确定性

百分比必须能追溯分子和分母：

```text
第 4 周留存 42%（84 / 200）
```

小样本可能造成大幅波动。图表应根据任务提供：

- 样本量；
- 置信区间或误差范围；
- 最小解释样本条件；
- 分群定义；
- 排除和缺失数量；
- 数据回补时间。

不要平均分位数。合并 P95 需要原始分布或可合并的直方图/摘要结构。

## 颜色与系列

颜色以外至少使用一种编码：

- 直接标签；
- 线型；
- 点形；
- 分面；
- 明确顺序；
- 交互选中后的文字详情。

图例顺序与图中系列顺序一致。颜色对比需在普通、悬停、选中和高对比模式检查。

## 等价数据表

等价表格与图表使用同一查询结果，不是人工维护副本。

表格包含：

- 系列名称；
- x 值；
- y 值；
- 单位；
- 样本量；
- 缺失原因；
- 时间窗；
- 筛选；
- 更新时间。

图表筛选、隐藏系列或改变时间窗后，表格与文本结论同步。

## 交互图表键盘模型

若每个数据点可聚焦：

1. 只让合理数量的点进入导航模型；
2. 方向键按时间或系列移动；
3. 焦点详情包含系列、x、y、单位和样本量；
4. Escape 关闭临时详情；
5. 缩放后仍能重置；
6. 虚拟化或 Canvas 绘制时提供独立可访问表示。

若数据点成千上万，逐点 Tab 不可用。提供摘要、筛选、搜索和数据表比制造巨大焦点序列更有效。

## Chart 计算验证

留存图验证：

```text
cohort_size = 200
retained_week_4 = 84
retention_week_4 = 84 / 200 = 42%
```

检查图上第 4 周点、tooltip、文本结论和数据表是否都为 42%，并同时显示样本量 200。

延迟图验证：

- P50、P95、P99 从同一时间窗分布计算；
- 不对多个实例的 P95 求平均；
- 合并使用原始样本或可合并直方图；
- 空时间桶显示缺失，不显示 0ms；
- 发布事件注释使用同一时区；
- 导出保留版本、地区与请求量。

## Chart 失败注入

1. y 轴 domain 不包含一个极大值。
2. 对数轴出现零。
3. 一条系列全部缺失。
4. 图例有两个同色系列。
5. tooltip 只能悬停取得。
6. 筛选后数据表仍是旧查询。
7. 小样本 cohort 大幅波动。
8. 动画在减少动态效果偏好下仍运行。
9. Canvas 图没有替代文本和表格。
10. 导出丢失单位和时间窗。

图表验收必须保存使用的原始数据夹具、计算脚本输出与查询版本。
