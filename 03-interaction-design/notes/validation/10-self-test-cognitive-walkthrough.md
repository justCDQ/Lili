# 原型自测中的认知走查

## 是什么

原型自测中的认知走查是一种用于检查交互假设、任务结果或边界状态的验证方法。它规定输入条件、观察对象、记录方式和结论适用范围。

## 为什么需要

设计者已知道信息位置和交互结果，容易把熟悉性误当可学性。自测认知走查要求显式限制评估者知识，并为每一步寻找界面证据。

验证的价值在于减少未经检查的设计假设。明确这一方法的适用问题、输入条件和输出，可以避免把单次观察、评估者判断或工具结果扩张为普遍结论，并为改版前后复测提供一致基线。

## 关键特性或原则

判断这一知识点时，应明确对象、触发条件、可见状态、用户可执行动作和完成标准。不同平台或任务风险会改变具体方案，不能只根据单张截图或单一控件名称下结论。

## 实际怎么使用

### 操作方法

1. 写下目标用户已知和未知的概念、术语与入口。
2. 从真实入口开始，不直接打开目标页面。
3. 每一步先回答用户当前目标，再检查正确操作是否可见、是否与目标关联。
4. 操作后只依据界面反馈判断是否进展，不引用设计意图。
5. 记录失败点、所需额外知识和最小修改。
6. 隔日或由未参与方案的人按同一任务复走，减少记忆影响。

### 证据格式

`步骤 / 用户目标 / 正确操作 / 可见线索 / 预期关联 / 系统反馈 / 判断 / 风险 / 修改`。

## 常见错误与边界

- 用“图标很常见”代替平台、标签或证据。
- 为原型缺失状态口头补充说明。
- 只检查能否完成，不检查为何选择。
- 自测只能发现风险，不能证明目标用户一定理解；关键假设仍需其他证据。

## 补充知识

单一验证方法通常只能覆盖部分问题。原则性检查适合发现风险，可运行测试适合确认行为，日志与埋点适合描述已发生事件；需要判断原因或推广范围时，应组合不同证据并记录反证、样本限制和版本时效。
