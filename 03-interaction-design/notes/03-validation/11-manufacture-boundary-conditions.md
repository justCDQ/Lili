# 制造边界条件进行验证

边界条件验证通过有计划地构造最小值、最大值、无效值、权限变化、网络故障、并发和生命周期交界，检查设计与实现是否仍满足任务和数据不变量。目标不是随机“搞坏页面”，而是从规则、状态和风险推导可重复测试。

## 什么是边界条件

边界条件位于允许与不允许、一个状态与另一个状态、一个责任系统与另一个系统的交界。例如：

- 文件大小正好等于上限与超过 1 字节；
- 截止日期等于今天与早于今天；
- 用户提交前有权限，提交时被撤销；
- 上传完成但服务端处理尚未开始；
- 两个编辑者从同一版本同时修改；
- 请求写入成功但响应在网络中丢失。

常规示例只证明常规条件。边界更容易暴露定义模糊、前后端规则不一致和恢复缺口。

## 从规则推导测试

```mermaid
flowchart LR
    R["业务、数据与平台规则"] --> D["识别阈值和状态交界"]
    D --> C["构造边界数据与环境"]
    C --> E["写预期不变量"]
    E --> T["执行与故障注入"]
    T --> O["核对 UI、请求和权威数据"]
    O --> G["修复与回归集合"]
```

先写预期不变量，避免看到实际结果后临时改变标准。

## 边界类别

### 数值和长度

- 最小、最小减一、最小加一；
- 最大、最大减一、最大加一；
- 零、负数、极大值、小数和单位转换；
- 空字符串、一个字符、最大长度和超长；
- Unicode、组合字符、表情、换行和双向文本。

字符数、字节数和视觉宽度不同。文件大小与传输限制也可能使用不同单位，规则必须明确。

### 集合

- 0、1、2、分页边界、批量上限；
- 全部有效、全部无效、混合有效；
- 重复、同名、缺失字段和无序；
- 选择跨页、筛选后隐藏和数据动态插入。

### 时间

- 当前时间、截止前后、午夜、月末、闰日；
- 时区、夏令时和本地/UTC 转换；
- 会话、链接和缓存过期前后；
- 长任务跨越权限或规则变更。

### 权限和身份

- 未登录、会话过期、只读、编辑者、管理员；
- 权限在页面打开后、提交前和处理中变化；
- 对象转移所有者、组织或可见范围；
- 不应泄露对象存在性的访问。

### 网络和系统

- 慢响应、断网、恢复、超时和响应乱序；
- 请求成功响应丢失；
- 第三方成功但本系统失败，或相反；
- 队列积压、异步任务卡住和重复投递；
- 客户端刷新、崩溃、后台和多标签页。

### 并发和版本

- 同字段冲突与不同字段修改；
- 删除与编辑同时发生；
- 重复点击、网络重试和多设备重放；
- 旧客户端读取新状态；
- 迁移中旧数据与新数据共存。

### 无障碍与布局

- 仅键盘、屏幕阅读器和语音输入；
- 200% 放大、320 CSS 像素、文本间距调整；
- 长翻译、高对比、减少动态和方向变化；
- 焦点目标被固定内容遮挡；
- 悬停内容在触屏和键盘下的替代。

## 组合策略

所有维度的笛卡尔积会爆炸。使用：

### 等价类

把会得到相同行为的输入归组，每组选代表值。例如合法日期、过早日期和格式非法日期。

### 边界值

对阈值取边界两侧值。例如上限 10 MB：9,999,999、10,000,000、10,000,001 字节，具体单位按实现规则。

### 风险配对

优先组合会放大风险的维度：大文件 + 弱网；高风险提交 + 响应丢失；离线修改 + 权限撤销；200% 放大 + 错误摘要。

### 状态转换覆盖

确保每条允许转换和禁止转换至少验证一次。比随机组合页面截图更有效。

### 生产事故回归

把真实 Issue、支持和日志中的条件固化为自动或手工回归数据。

## 故障注入安全

- 优先测试环境、合成数据和专用账户；
- 写操作使用幂等标识和清理策略；
- 不在生产制造支付、删除、权限或资源耗尽故障；
- 网络拦截只针对测试请求，防止影响其他系统；
- 日志、截图和 HAR 脱敏；
- 明确停止条件和恢复步骤。

## 完整案例：文件上传边界矩阵

### 规则和具体输入

```text
允许类型：PDF
最大文件：10,000,000 字节
同一项目文件名可重复，但内容哈希相同视为重复上传
角色：项目编辑者
上传完成后服务端异步扫描与解析
```

固定文件：

```text
F1 empty.pdf：0 字节
F2 exact.pdf：10,000,000 字节，合法 PDF
F3 over.pdf：10,000,001 字节
F4 renamed.exe.pdf：扩展名 PDF，内容不是 PDF
F5 duplicate.pdf：内容与现有文件哈希相同
F6 long-name-…pdf：文件名 260 个 Unicode code point
```

### 测试矩阵

| ID | 输入/环境 | 预期 |
| --- | --- | --- |
| B-01 | F1 | 拒绝空文件，说明要求，不发起处理 |
| B-02 | F2 | 允许传输，最终由扫描与解析决定 |
| B-03 | F3 | 在可判断阶段拒绝，服务端再次校验 |
| B-04 | F4 | 不能只信扩展名，服务端拒绝内容 |
| B-05 | F5 | 提示重复并链接现有文件，默认不创建副本 |
| B-06 | F6 + 320px | 名称可换行/截断并可获得全名，操作不溢出 |
| B-07 | F2 + 45% 断网 | 暂停或失败可恢复，不创建重复会话 |
| B-08 | F2 + 上传后撤权 | 服务端处理/写入按明确授权规则处理 |
| B-09 | F2 + 响应丢失 | 查询同一会话，不创建重复版本 |
| B-10 | F2 + 扫描失败 | 区分上传成功和处理失败，可安全重试 |

### 逐步处理 B-09

1. 客户端创建上传意图 `UP-9001`。
2. 分块上传完成，服务端创建处理任务 `PROC-81`。
3. 最终响应在网络中丢失，客户端超时。
4. 界面显示“正在确认上传结果”，不显示确定失败。
5. 客户端按 `UP-9001` 查询，取得 `PROC-81`。
6. 页面恢复“文件已上传，正在扫描”，最终进入可查看。

### 可观察输出

```text
上传意图：UP-9001
文件版本：V3（唯一）
处理任务：PROC-81
传输：完成
扫描：完成
解析：完成
重复文件版本：0
```

### 失败分支

- 查询接口也不可用：保留未知状态和会话 ID，允许稍后查看，不生成新意图。
- 服务端在上限判断使用 MiB、客户端使用 MB：B-02 会暴露不一致，统一规则与单位。
- 长文件名截断后两个文件视觉相同：显示可区分尾部或完整名称访问方式。
- 扫描判定恶意：隔离文件、不提供下载、显示安全范围内的错误并记录审计。

### 键盘与辅助技术

选择、取消、替换、查看重复和重试均可键盘操作；进度阶段变化可感知但不过度播报；错误以文本描述；焦点在失败后进入错误摘要或保持在相关控件。

### 验证证据

- 客户端与服务端均记录字节规则；
- 数据库只有一个版本和一个处理任务；
- 网络记录显示相同上传意图；
- 320 CSS 像素无页面横向溢出；
- 屏幕阅读器读出文件名、阶段和错误。

## 常见错误与修正

- 随机输入很多值却不对应规则：从阈值和状态转换推导。
- 只测前端校验：绕过客户端验证服务端不变量。
- 错误测试没有检查任何副作用是否已发生。
- 重试只看界面，不核对重复数据。
- 在生产制造危险故障。
- 边界矩阵组合无限：用等价类、风险配对和转换覆盖。
- 只测数据边界，不测权限、时间、网络和无障碍。
- 修复后删除失败用例：固化为回归。

## 可执行步骤

1. 收集业务、数据、权限、平台与无障碍规则。
2. 标出阈值、生命周期和系统责任边界。
3. 为每个边界写两侧值、预期状态和不变量。
4. 使用等价类、风险配对和转换覆盖缩减组合。
5. 准备合成数据、故障开关、幂等与清理。
6. 观察界面、DOM、请求、日志和权威数据。
7. 验证取消、重试、刷新、跨端与辅助技术。
8. 把真实失败条件加入持续回归。

## 练习与完成标准

为“重置密码链接”制造边界条件。

完成时应满足：

- 覆盖令牌刚创建、刚过期、重复使用和已撤销；
- 覆盖账户不存在、安全不泄露、会话过期和密码规则边界；
- 两个设备同时使用同一令牌；
- 写入成功响应丢失时能确认结果；
- 键盘、屏幕阅读器、200% 放大和窄屏可完成；
- 服务端授权与令牌不变量有权威验证；
- 使用合成账户，测试后完成清理并固化回归。
