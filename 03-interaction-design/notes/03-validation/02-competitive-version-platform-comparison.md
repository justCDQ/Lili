# 竞品、版本与平台对比验证

对比验证使用同一任务、固定数据和明确完成标准，检查不同产品、旧新版本或 Web/iOS/Android 的流程、状态、错误和结果差异。它用于验证设计假设和发现边界，不用于复制竞品或用功能清单给产品排名。

## 三类对比的不同问题

| 对比类型 | 主要目的 | 关键变量 |
| --- | --- | --- |
| 竞品对比 | 理解同一任务的不同方案与取舍 | 用户、业务规则、对象模型、定价和技术能力 |
| 版本对比 | 判断改动解决了什么、引入什么回归 | 构建、迁移、实验、旧数据与兼容 |
| 平台对比 | 验证跨端结果一致与平台操作合理 | 输入、返回、通知、文件、辅助技术和布局 |

三类可以组合，但报告必须区分。一个产品在 iOS 的方案不能直接与另一个产品 Web 方案比较后归因于产品能力。

## 可比实验设计

```mermaid
flowchart LR
    T["定义同一任务"] --> I["准备等价输入和账户"]
    I --> P["记录产品、版本与平台"]
    P --> R["分别执行主路径和故障"]
    R --> M["对齐状态、输出和指标"]
    M --> E["解释条件差异"]
    E --> D["形成待验证设计决策"]
```

### 固定任务

任务声明包含用户目标、角色、对象、触发、前置条件和完成标准。“比较项目管理”太宽，应改为“编辑者在截止前从通知进入任务并修改截止日期”。

### 等价输入

准备相同数量、长度、错误和权限条件。不同产品对象模型不同，可使用语义等价数据，并明确不能完全对齐的部分。

### 记录环境

- 产品、版本、构建和观察时间；
- 账户层级、订阅、角色和组织设置；
- 平台、操作系统、浏览器、视口与缩放；
- 语言、地区、时区和网络；
- 功能开关、测试组与新旧数据。

## 对齐维度

### 入口与范围

检查通知、深链、搜索、导航与恢复入口。登录或升级后是否返回原任务，属于入口行为。

### 对象与术语

记录每个产品怎样命名对象、状态和动作。相同“归档”可能有不同保留和权限结果，不能只对齐名称。

### 步骤与判断

记录需要理解、输入、选择、等待和复核的节点，不只计算点击。平台原生文件选择和系统分享也属于流程。

### 状态与恢复

至少比较初始、加载、空、成功、失败、无权限、离线、过期、取消、重试和并发中的相关状态。

### 结果

分别验证界面结果、对象状态、业务状态、通知或外部副作用，以及刷新与跨端一致性。

### 无障碍和输入

比较键盘、触控、屏幕阅读器、文字放大、方向和系统返回。不同平台遵循不同惯例可以合理，但任务结果与关键规则必须一致。

## 版本对比的特殊检查

### 新旧数据

分别使用新建对象和旧版本迁移对象。新版本在全新数据上成功，不能证明迁移后的权限、默认值和状态正确。

### 渐进发布

更新日志的发布日期不代表所有账户同时获得功能。记录实验组、地区、应用商店发布和服务端开关。

### 回归范围

对改动节点检查上游入口、下游结果、错误和辅助技术，而不是只看新组件。状态名或数据模型变化可能影响通知、搜索和导出。

### 回滚

确认新版本产生的数据能否被旧版本读取，移动端未升级时怎样处理。无法回滚的迁移需要明确发布控制和恢复方案。

## 平台对比的特殊检查

### 导航与返回

Web 浏览器前进后退、iOS 导航栈、Android 系统返回和手势可能不同。验证取消、未保存、深链和对话框关闭的语义，而不是要求布局完全相同。

### 输入与文件

桌面键盘鼠标、移动触控、软键盘、相机、系统文件选择和分享能力不同。目标大小、焦点、遮挡和权限弹窗需要平台实测。

### 通知与后台

推送权限、后台限制和系统通知设置会改变入口与结果。产品不能把“服务器已发送推送”写成“用户已收到”。

### 辅助技术

Web 屏幕阅读器、VoiceOver、TalkBack 和平台可访问性 API 行为不同。自动扫描不能替代真实操作。

## 完整案例：Web、iOS、Android 的文件上传恢复

### 固定输入

```text
任务：向项目 P-204 上传设计稿并确认服务端处理完成
文件：roadmap-v3.pdf，18 MB
角色：项目编辑者
网络：上传 45% 时断开 10 秒，然后恢复
完成：文件版本存在、处理状态完成、跨端可查看、无重复版本
```

### 环境

```text
Web：Chrome，1280×720 CSS px，build 2026.07.15
iOS：应用 6.4.1，iOS 19，VoiceOver 另行测试
Android：应用 6.4.1，Android 16，TalkBack 另行测试
时区/语言：Asia/Shanghai，简体中文
```

### 观察结果

| 节点 | Web | iOS | Android |
| --- | --- | --- | --- |
| 选择文件 | 浏览器文件选择器 | 系统文件与云盘 | 系统文件选择器 |
| 断网 | 进度暂停，恢复后续传 | 状态转“等待网络”，允许后台 | 进度失败，要求手动重试 |
| 离开页面 | 任务中心可恢复 | 后台继续并发通知 | 离开后任务保留但不自动重试 |
| 处理阶段 | “已上传，处理中” | 同义状态 | 显示“上传完成”，需核实是否误指最终完成 |
| 最终结果 | 版本 V3 可查看 | V3 可查看 | V3 可查看 |

### 进一步验证

1. 网络记录显示三端使用同一上传会话 ID 和文件校验值。
2. Android 的“上传完成”出现时，服务端状态仍为 `processing`，3 秒后才为 `ready`。
3. 关闭 Android 应用后重新打开，任务详情能恢复，但通知未说明处理结果。
4. 使用同一上传会话重试没有创建 V4；新会话上传同一文件则按业务规则提示重复。

### 输出结论

- 三端数据结果一致，Web 和 iOS 对传输与处理阶段表达明确。
- Android 文案把传输完成与处理完成混合，是当前版本可复现事实。
- Android 不自动重试可能来自后台限制或产品选择，不能仅根据差异判定错误；但手动恢复与结果通知需要明确。

### 失败分支

- 文件权限被系统撤销：移动端应说明无法继续并允许重新选择，不把它当服务器错误。
- 项目权限在断网期间撤销：恢复后服务端拒绝写入，三端都不得显示成功。
- 处理服务失败：文件可显示“处理失败”，允许替换或重试处理，不重复上传。
- 应用旧版本 6.3 打开 V3：验证是否识别新处理状态；若未知，应安全显示只读和升级说明。

### 无障碍与响应式验证

- Web 仅用键盘选择文件、取消、恢复和进入结果。
- VoiceOver 与 TalkBack 能获知传输、等待网络、处理和完成，不持续播报每个百分比。
- 200% 放大和软键盘出现时，取消与状态不被遮挡。
- 系统返回不会无说明地停止后台任务。

## 常见错误与修正

- 不同订阅和权限直接比较：建立等价账户基线。
- 用截图比较动态流程：实际执行、录制状态和请求。
- 把点击数作为唯一指标：加入判断、等待、错误和恢复。
- 将平台差异都当不一致：区分平台惯例与结果规则。
- 新版本只测新对象：加入旧数据与未升级客户端。
- 竞品方案直接复制：在自己的业务、数据和工程条件下验证。
- 观察日期不记录：无法区分修复、实验和分阶段发布。

## 可执行步骤

1. 定义同一任务、角色、输入和完成标准。
2. 准备等价账户和安全测试数据。
3. 记录产品、构建、平台、语言、网络和开关。
4. 分别执行主路径、失败、权限、取消和恢复。
5. 对齐入口、对象、步骤、状态、结果和无障碍。
6. 将事实差异与约束解释分开，列出反证。
7. 对版本加入迁移、旧客户端和回滚检查。
8. 把可取机制做成自己的原型，用同一任务重新验证。

## 练习与完成标准

比较两个产品和一个旧版本的“导出数据”流程。

完成时应满足：

- 固定数据范围、格式、权限、设备和完成标准；
- 记录生成、处理、通知、下载和过期状态；
- 覆盖无数据、无权限、生成失败、链接过期和重复请求；
- 版本对比包含旧导出任务与新版本读取；
- 平台对比包含键盘、系统返回和辅助技术；
- 每个差异区分事实、约束解释和未知项；
- 最终结论没有按功能数量或个人偏好排名。
