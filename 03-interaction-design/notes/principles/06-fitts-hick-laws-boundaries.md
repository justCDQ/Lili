# Fitts 定律与 Hick–Hyman 定律：公式、应用与边界

Fitts 定律描述快速指向任务中移动时间与目标距离、有效宽度的关系；Hick–Hyman 定律描述选择反应任务中反应时间与选择信息量的关系。它们来自受控实验，是预测特定条件下平均表现的模型，不是“按钮越大越好”或“选项越少越好”的通用界面法则。

## 前置概念

- **指向任务**：使用鼠标、触控、笔或其他指针移动并激活目标。
- **选择反应任务**：在已知刺激与响应集合中，根据出现的刺激选择对应响应。
- **平均时间模型**：公式预测一组实验条件下的平均值，不保证单个用户、单次操作时间。
- **拟合参数**：公式中的截距和斜率需要用具体设备、任务和人群的数据估计，不能从其他产品直接复制。

## Fitts 定律

### 常用公式

人机交互研究常用 Shannon 形式：

```text
MT = a + b × ID
ID = log2(D / W + 1)
```

变量含义：

| 变量 | 含义 | 单位与边界 |
| --- | --- | --- |
| `MT` | 完成指向动作的平均移动时间 | 通常为毫秒 |
| `a` | 与任务、设备、人群有关的截距 | 由数据拟合，不是固定常数 |
| `b` | 每增加一 bit 难度所增加的时间 | 通常为毫秒/bit，由数据拟合 |
| `D` | 起点到目标中心沿运动方向的距离 | 与 `W` 使用相同空间单位 |
| `W` | 沿运动方向允许成功命中的有效宽度 | 是命中区域，不一定等于视觉宽度 |
| `ID` | 难度指数 | bit，因为使用以 2 为底的对数 |

Fitts 1954 原始实验使用不同的难度表达；现代 HCI 常使用上面的 Shannon 形式。引用研究或比较数据时必须说明采用哪一公式和测量方法。

### 公式怎样工作

- 距离 `D` 增加，难度指数与平均移动时间增加。
- 有效宽度 `W` 增加，难度指数与平均移动时间降低。
- 关系经过对数变换，因此距离翻倍不会简单让时间翻倍。
- 只有 `D/W` 的比值相同且其他条件相近时，模型才预测相同难度。

### 计算示例

假设同一设备实验拟合得到 `a = 80 ms`、`b = 120 ms/bit`。这两个数只用于计算示例。

目标 A：`D = 400 px`，`W = 40 px`。

```text
ID_A = log2(400 / 40 + 1)
     = log2(11)
     ≈ 3.459 bit

MT_A = 80 + 120 × 3.459
     ≈ 495 ms
```

目标 B 保持距离 400 px，把有效宽度增至 80 px：

```text
ID_B = log2(400 / 80 + 1)
     = log2(6)
     ≈ 2.585 bit

MT_B = 80 + 120 × 2.585
     ≈ 390 ms
```

在该拟合模型中，预测差约 105 ms。这不是对任何网页按钮的固定收益；触控、手部移动、视觉搜索、目标识别和错误代价可能改变结果。

### 界面应用

- 高频、主要和时间敏感的指针目标应有足够命中区域。
- 增大透明点击区可以提高有效宽度，但相邻目标不能重叠或产生误触。
- 屏幕边缘可在某些桌面指针条件下限制越过目标，但浏览器窗口边缘、触屏和系统手势会改变这一优势。
- 目标分组应考虑起始位置；只比较按钮自身大小而不比较实际移动路径不完整。
- 拖放包含取得目标、移动与放置多个阶段，应分别分析，而且必须提供非拖动等价操作。

WCAG 2.2 的 2.5.8 Target Size (Minimum) 是可访问性合规要求：目标至少 24×24 CSS 像素，或满足列出的间距、等价、行内、用户代理与必要性例外。它不是由 Fitts 公式直接推导出的“最佳尺寸”，也不保证目标在具体任务中足够易用。

## Hick–Hyman 定律

### 等概率选项的常见形式

```text
RT = a + b × log2(n + 1)
```

| 变量 | 含义 | 边界 |
| --- | --- | --- |
| `RT` | 从刺激出现到作出选择响应的平均反应时间 | 不一定包含指向与执行时间 |
| `a` | 基础反应与任务截距 | 由实验拟合 |
| `b` | 每 bit 信息量增加的反应时间 | 由实验拟合 |
| `n` | 可区分且等概率的响应数量 | 不是页面上所有可见元素的简单计数 |

Hyman 的实验进一步考察了选项概率不相等和序列依赖。更一般的分析使用信息熵：

```text
H = -Σ pᵢ × log2(pᵢ)
RT = a + b × H
```

当所有 `n` 个选项等概率时，`H = log2(n)`。不同文献的公式、常数和实验设计不同，不能混用 `log2(n)` 与 `log2(n+1)` 后直接比较数值。

### 计算示例

假设一次等概率选择任务拟合得到 `a = 250 ms`、`b = 150 ms/bit`。

四个选项：

```text
RT_4 = 250 + 150 × log2(4 + 1)
     ≈ 250 + 150 × 2.322
     ≈ 598 ms
```

八个选项：

```text
RT_8 = 250 + 150 × log2(8 + 1)
     ≈ 250 + 150 × 3.170
     ≈ 726 ms
```

模型预测增加约 128 ms。若八个选项具有熟悉分组、明显频率差异或用户可以直接搜索，这个简单等概率模型不再准确描述实际任务。

### 界面应用

- 减少当前决策中无关或不合法的选项，而不是任意删除功能。
- 使用清晰类别、默认值、搜索和最近使用，让选项概率与结构可利用。
- 相似选项需要可区分名称和比较维度；只减少数量不能解决含义重叠。
- 高频操作可置于直接可见位置，低频操作放入可发现菜单，但要测量发现成本。
- 复杂决定可能需要更多信息、复核和时间；强行减少步骤会增加错误风险。

## 两个模型不能混为一谈

```mermaid
flowchart LR
    S["发现候选操作"] --> H["Hick–Hyman：选择哪一个"]
    H --> F["Fitts：移动并命中目标"]
    F --> E["执行、等待与理解结果"]
```

真实任务时间还包括视觉搜索、阅读、记忆、输入、系统延迟和错误恢复。Hick–Hyman 主要对应选择反应，Fitts 主要对应指向动作；二者都不能单独预测端到端任务时间。

## 完整案例：移动端订单行操作

### 具体输入与证据

订单列表每行当前显示六个图标操作：查看、复制编号、联系客户、退款、取消、更多。图标仅在悬停显示，但移动端没有稳定悬停；触控日志显示“取消”误触后立即返回的比例高。日志只能说明发生了回退，不能证明全部原因。

实际任务频率样本：

```text
查看详情：62%
复制编号：20%
联系客户：10%
退款：4%
取消：1%
其他：3%
```

### 分析过程

1. “查看详情”可让整行或明确链接承担，减少独立小图标。
2. “复制编号”频率高且低风险，保留为有文字或可靠名称的直接操作。
3. “联系客户、退款、取消”进入“更多操作”菜单；退款与取消在菜单中分离，并显示文字结果。
4. 危险操作不能只因低频藏起来后直接执行；选中后进入对象明确的复核步骤。
5. 每个触控目标检查 CSS 像素尺寸和相邻间距，同时保留键盘等价路径。

### 输出设计

```text
订单 #A1024             待发货
客户：李晓              ¥328.00
[查看详情] [复制编号] [更多操作]
```

“更多操作”展开：联系客户、退款、取消订单。菜单项使用文字，当前不合法操作隐藏或禁用并解释；键盘使用标准菜单或普通按钮列表的对应模式，不能混合语义。

### 定量检查

假设旧取消图标有效宽度 20 px，新菜单项有效高度 44 px，指针起点到目标中心距离分别为 120 px 和 180 px：

```text
旧 ID = log2(120 / 20 + 1) = log2(7) ≈ 2.807
新 ID = log2(180 / 44 + 1) ≈ log2(5.091) ≈ 2.348
```

在相同拟合参数前提下，新目标的指向难度指数较低，但打开菜单增加了一个选择步骤。最终是否更好要同时测量误触、任务时间、发现率和危险操作复核，而不是只依据公式。

### 状态、键盘与失败分支

- 菜单触发按钮有可访问名称和展开状态；关闭后焦点回到触发按钮。
- 列表滚动或数据刷新时，打开菜单要锚定正确订单或安全关闭。
- 订单状态被他人更新后，服务端重新校验退款与取消资格。
- 请求超时不能显示已取消；用订单详情的权威状态验证。
- 若把低频操作收入菜单导致退款发现率明显下降，检查命名、入口与任务优先级，而不是立刻增加更多图标。

### 验证方案

1. 用相同设备和测试数据比较旧、新方案。
2. 分别测量查看、复制、退款和取消的发现率、正确率、误触和完成时间。
3. 记录起始位置、目标尺寸和输入设备，避免混合不可比数据。
4. 仅用键盘和屏幕阅读器完成所有操作。
5. 在窄屏、200% 放大、长订单号和动态刷新条件下复测。

## Fitts 模型的边界

- 目标必须可见且可识别；视觉搜索时间不在简单公式中。
- 有效宽度沿运动方向计算，二维不规则目标需要更精确模型。
- 不同设备、姿势、人群和错误率需要重新拟合。
- 触摸目标受手指遮挡、握持、拇指活动范围和系统手势影响。
- 公式预测速度，不直接包含误触成本、学习和可访问性。
- 键盘、语音、开关控制和屏幕阅读器导航不是同一指向任务。

## Hick–Hyman 模型的边界

- 选项数量不等于有效信息量；概率、分组、熟悉度和刺激—响应兼容性会影响结果。
- 模型来自受控选择反应任务，不能直接解释阅读、价值判断或开放式决策。
- 隐藏选项会增加发现和导航步骤，不能只按当前屏幕选项数计算。
- 已知目标的菜单搜索与不知道选什么的决策不是同一种任务。
- 高风险决定需要足够信息和确认，时间更短不一定结果更好。

## 可执行应用步骤

1. 明确分析的是指向、选择，还是包含搜索与理解的完整任务。
2. 记录输入设备、起点、目标命中区域、距离、选项及其概率。
3. 选择公式并说明版本、单位与拟合参数来源。
4. 计算难度或信息量，只作为候选方案比较的一项证据。
5. 检查 WCAG 目标尺寸、键盘等价、名称和焦点要求。
6. 用真实任务测量正确率、误触、发现率、完成时间和恢复。
7. 对设备、人群和任务条件变化重新验证，不外推单一拟合结果。

## 常见错误与修正

- 把视觉图标大小当作有效命中宽度：检查真实点击区域。
- 只增大目标却让相邻点击区重叠：同时验证间距与误触。
- 用 Fitts 定律解释用户找不到按钮：先检查可见性和识别。
- 用 Hick 定律要求菜单最多七项：没有固定通用上限，应分析信息量和结构。
- 把选项移入多层菜单后宣称选择更快：必须计入发现与导航。
- 复制论文或其他产品的 `a`、`b` 参数预测本产品：应使用本任务数据拟合。
- 只比较平均时间，不报告错误率和数据分布。

## 练习与完成标准

比较桌面工具栏的两个方案：8 个直接按钮；4 个直接按钮加一个含 4 项的菜单。

完成时应满足：

- 记录每项任务频率、风险、输入设备与起始位置；
- 测量目标有效尺寸和距离，并用同一 Fitts 公式计算；
- 说明 Hick–Hyman 模型中选项是否可视为等概率；
- 不把菜单打开、视觉搜索和阅读时间排除在最终任务测量外；
- 覆盖键盘、屏幕阅读器、窄屏和 200% 放大；
- 报告正确率、误触、发现率、时间中位数或分布，而不只给平均值；
- 用数据说明选择，并明确不能外推的条件。
