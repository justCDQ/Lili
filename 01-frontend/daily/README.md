# 每日前端项目

每天完成一个小而可运行的前端项目。重点不是连续打卡，而是用项目验证当天学习的一个概念；复杂项目可以连续多天，但每天必须有独立增量和证据。

## 可以做什么

- HTML：文章结构、表单、表格、Dialog、可访问导航。
- CSS：布局、组件、响应式、动效、主题、设计稿还原。
- JavaScript：DOM 交互、小游戏、数据处理、异步和 API。
- TypeScript/框架：组件、表单、路由、状态、请求和测试。
- 工程深入：性能实验、DevTools 排障、构建插件和 Codemod。

## 记录方式

1. 复制 [_template.md](_template.md)。
2. 按 `YYYY/MM/YYYY-MM-DD-topic.md` 保存，例如 `2026/07/2026-07-16-accessible-dialog.md`。
3. 代码较小时可放在记录旁的同名目录；完整项目放独立仓库并在记录中链接。
4. 至少保留运行地址、仓库/代码、截图、测试或 Profile 中的一种证据。

## 完成标准

- [ ] 有一个明确、足够小的学习目标。
- [ ] 能运行或能复现，不只是复制代码。
- [ ] 记录关键原理、遇到的问题和自己的取舍。
- [ ] 写出下一次如何扩展或改进。
