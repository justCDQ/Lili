# 三年成长规划

## 总体原则

- 每季度一个主方向。
- 每月一个可交付成果。
- 每半年一个综合项目。
- 每年一次能力盘点。
- 所有学习都必须有代码、数据、文档或用户反馈作为证据。

---

# 第一年：前端、产品、交互与 AI 应用

## Q1：前端底层与性能

学习：Web 基础、浏览器、DevTools、内存、性能、可访问性。

产出：性能实验室、真实项目性能报告、性能预算和回归检查。

## Q2：产品与交互

学习：用户问题、JTBD、信息架构、User Flow、模式库、可用性测试。

产出：三次访谈、30 天产品拆解、交互模式库 V1、一个复杂 B 端原型。

## Q3：AI 应用基础

学习：模型 API、Prompt、Context、Structured Output、Streaming、AI UX。

产出：AI Chat、结构化抽取服务、Prompt Evaluation。

## Q4：RAG 与 AI 工作区

学习：文档解析、Chunk、Hybrid Search、Rerank、Citation、Artifact。

产出：RAG 文档助手、AI 工作区原型、RAG 评估报告。

---

# 第二年：后端、数据与分布式

## Q1：Go、Linux 与 API

产出：企业任务管理 API，包含 Auth、RBAC、多租户、OpenAPI 和测试。

## Q2：数据库与缓存

产出：订单系统数据模型、索引与事务报告、Redis 缓存实验。

## Q3：消息与分布式

产出：支持重试、死信、幂等、恢复和指标的分布式任务系统。

## Q4：云原生与可观测性

产出：Docker/Kubernetes 部署、CI/CD、OpenTelemetry、Dashboard 和故障演练报告。

# 长期补充能力

## 英语与信息检索

- 阅读官方文档、RFC、论文和 GitHub Issue/PR。
- 优先使用一手资料。
- 建立自己的英文技术术语库。

## 写作与表达

- RFC、ADR、架构文档、复盘、技术博客和分享。
- 训练把复杂问题讲清楚，而不是堆术语。

## 商业与财务

- SaaS、收入、成本、毛利、CAC、LTV、ROI、现金流基础。
- 技术方案不仅评估性能，也评估开发、维护和基础设施成本。

## 数据分析与实验

- SQL、漏斗、留存、Cohort、A/B Test、指标体系和实验偏差。

## 项目管理与领导力

- Roadmap、里程碑、风险、依赖、技术债、跨团队推动、指导他人和做决策。

## 安全与隐私

- OWASP、身份、权限、多租户、数据保存、删除、审计和供应链安全。

## 个人品牌与影响力

- GitHub、开源、博客、演讲、社区贡献和长期主题建设。

## 健康与可持续学习

- 睡眠、运动、注意力管理。
- 每季度做减法，避免长期超负荷。
- 把学习计划建立在可持续节奏上，而不是短期冲刺。
