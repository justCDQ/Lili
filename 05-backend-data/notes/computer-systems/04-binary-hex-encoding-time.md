# 二进制、十六进制、字符编码、时间与时区

## 是什么

二进制以位表示数据；十六进制每位对应 4 bit，便于显示字节。字符编码把字符映射为字节；UTF-8 是 Unicode 的可变长编码。时间点应以 UTC 时间戳或带偏移的 RFC 3339 表示；时区是包含历史和夏令时规则的地区标识。

```go
b := []byte("狸力")
fmt.Printf("%x\n", b)
t, err := time.Parse(time.RFC3339, "2026-07-16T10:00:00+08:00")
if err != nil { return err }
fmt.Println(t.UTC().Format(time.RFC3339))
```

## 关键特性或规则

- 字节数、字符数和显示宽度不同。
- 网络协议明确字节序；多字节整数常用大端序。
- 存储事件时间点用 UTC，展示时转换用户时区；日历日期可能没有时间点含义。
- 持续时间使用单调时钟测量，不用墙上时钟相减。
- 不手写夏令时偏移，使用 IANA 时区数据库。

## 常见错误与边界

Unix 时间戳通常不编码时区；闰秒和系统时钟调整会影响假设；JavaScript Date 内部毫秒精度且解析模糊字符串有兼容风险。跨系统必须约定单位与格式。

## 为什么需要

后端程序直接依赖操作系统资源和开发工具链。理解这些对象的生命周期、权限和性能指标，才能从进程、文件、时间或资源层定位故障，并保证本地与 CI 的操作可复现。

## 实际怎么使用

按本文命令建立一个最小实验，先记录工作目录、用户、环境、进程和输入规模，再执行操作并保存退出码、日志与耗时。主动制造权限不足、路径错误、资源饱和或时区差异，确认诊断步骤能定位到具体层。

## 补充知识

容器和 CI 会改变命名空间、用户、工作目录、可用 CPU/内存和时区。任何依赖本机默认值的行为都应显式配置，并在目标运行环境再次验证。
