# 后端与数据模块学习笔记

每完成一个路线图模块，就在这里沉淀可复用的系统知识。复制 [后端与数据笔记模板](../notes-template.md)。算法日练放在 `daily/`，能够迁移到真实系统的结构和模式再总结到这里。

建议结构：

```text
notes/
├── programming-go/
├── algorithms/
├── linux-network/
├── api-database/
├── cache-messaging-storage/
├── distributed-systems/
└── cloud-sre-security/
```

每份笔记应包含需求与规模假设、原理、最小实验、测试或 Profile、故障模式、恢复方式、成本和架构取舍。
