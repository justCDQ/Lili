# 每日算法题（JavaScript / Go）

每天完成一道与当前阶段匹配的算法或数据结构题。目标是训练问题建模、复杂度意识、边界处理和清晰实现，不追求无意义的题量或竞赛技巧。

## 语言策略

- 零基础阶段先用 JavaScript 表达思路，降低语法负担。
- 同一题再用 Go 重写，练习 Slice、Map、Struct、Pointer、Error 和测试。
- 已熟悉两种语言后可以交替使用；关键题保留双语言版本并比较差异。

## 学习顺序

1. 复杂度、数组、字符串、哈希、栈、队列。
2. 双指针、滑动窗口、二分、递归和排序。
3. 链表、树、堆、Trie、图与并查集。
4. 回溯、贪心、动态规划和真实系统数据结构。

## 记录方式

1. 复制 [_template.md](_template.md)。
2. 按 `YYYY/MM/YYYY-MM-DD-problem.md` 保存，代码放同名目录。
3. 先写暴力解和复杂度，再优化；失败思路也要保留。
4. 每周重做一道旧题，不查看答案；每月实现一个 LRU、限流器、调度器等真实数据结构。

## 完成标准

- [ ] 能用自己的话重述问题、输入、输出和约束。
- [ ] 覆盖空值、最小值、重复、极端规模等边界。
- [ ] 说明时间与空间复杂度以及为何选择当前结构。
- [ ] 至少包含可运行样例或自动化测试。
